# 宫格故事版拼版

## 核心定义

宫格故事版不是把若干画面排在一起。每一格必须同时交付“决定性画面”和“导演执行信息”。纯黑白草图只承担构图、人物位置、动作相位和景别；镜号、时码、运动箭头、声音与衔接必须由确定性拼版程序添加，禁止依赖生图模型书写文字。

黑白草图允许简化演员五官、服装纹理和场景材质，但不得省略：人物身份区分、左右位置、身体朝向、视线目标、关键道具、动作阶段、前中后景和空间地标。

## 工作流

1. 完成并校验 BOARD Markdown。
2. 单镜图片以完整镜号命名，如 `S01-B01-C01.png`。
3. 解析 Markdown：
   `python3 scripts/parse_storyboard_md.py board.md --image-dir shots --out manifest.json`
4. 必要时在 manifest 中补 `character_direction` 与 `camera_direction`，值为 `left` 或 `right`。
5. 拼版：
   `python3 scripts/compose_storyboard_grid.py manifest.json --preset vertical-6 --out-dir output`
6. 验收：
   `python3 scripts/validate_storyboard_grid.py manifest.json --output-dir output`

## 版式选择

- `vertical-6`：9:16 单镜，2列×3行；默认推荐。
- `classic-6`：横版单镜，3列×2行。
- `director-5`：导演五格风格；当前实现使用六格画布，最后一格可留空作为图例区。
- `black-white-director-5`：黑白导演五格宫格，固定“上三格、下两格、右下图例”。适用于快速参考和审查镜头方向。样式参数见 [../assets/black-white-director-grid-style.json](../assets/black-white-director-grid-style.json)，视觉样板见 [../assets/black-white-director-grid-style-reference.png](../assets/black-white-director-grid-style-reference.png)。

## 黑白导演五格默认样式

用户要求“黑白宫格图、黑白故事版、草图参考”且未要求详细文字时，默认使用 `black-white-director-5`：

- 横向暖白画布，上排三格，下排两格；右下角保留图例；
- 每格上方只显示稳定专业镜号与时码，例如 `S01-B01-C01  0.0–3.0s`；
- 禁止另设全片顺序递增号；Beat 改变时严格跟随 BOARD 镜号，从该 Beat 的 `C01` 重新开始；
- 画内实线箭头表示人物运动；
- 画内虚线箭头表示摄影机运动；
- 实线弧箭头表示弧移；虚线弧箭头表示俯仰；
- 右下角图例只显示 `Character Movement / Camera Movement / Arc Motion / Tilt`；
- 不显示轴线、台词、道具、衔接、镜头功能或其他段落文字；
- 精确镜号和时码由程序绘制，生图模型只生成无文字黑白画面与必要运动箭头；
- 若一组超过五镜，分页，不压缩成小格，不删镜。

本样式与“详细导演卡片版”互斥。用户说“其他都不用、参考图这种标注”时，必须选择本样式，不得自动增加说明文字块。

### 样式继承与禁止降级

- 样式一经用户确认，在同一任务后续分页、重绘、补镜和继续生成中自动继承；“继续”不是重新选择样式的授权。
- 不得用通用 `director-5`、`classic-6`、详细卡片版或纯图片页代替 `black-white-director-5`。
- 黑白快速参考默认允许一次生成整张五格无文字画面；随后必须由程序添加精确标注。只有正式资产级输出才逐镜生成。
- 生图阶段不得包含边框标题、镜号、时码、箭头、图例或任何文字；这些全部属于确定性标注层。
- 整页无文字画面是 `ROUGH-GRID` 的合法画面层，但未经过程序标注前不得称为完成宫格。
- 用户要求快速、草图、参考或“直接出宫格”时，不得逐镜调用生图；每页只生成一次画面源，且不向用户展示中间素材。

### ROUGH-GRID 快速生产顺序

1. 从 BOARD 锁定本页 1–5 个稳定镜号与准确时码。
2. 一次生成整张“上三下二、右下留白”的无文字黑白画面源；不得输出中间源图给用户。
3. 程序添加格顶 `Sxx-Bxx-Cxx  start–end`、运动箭头与右下四项英文图例。
4. 对照视觉样板逐项验收；全部通过后才可称为正式宫格页。

### PRECISION-GRID 逐镜生产顺序

1. 逐镜生成并校验人物身份、轴线、动作和道具。
2. 按完整镜号保存源图并写入 manifest。
3. 使用固定模板确定性拼版和标注。
4. 仅在用户明确需要正式资产级精度时启用。

### 零容差版式验收

以下任一情况出现，输出判为失败，不得交付：

- 不是上排三格、下排两格、右下图例；
- 格顶缺少或误写 `Sxx-Bxx-Cxx  0.0–3.0s` 格式的稳定镜号时码；
- 使用全片顺序递增的 `C01、C02……` 代替 BOARD 稳定镜号；
- 人物运动与摄影机运动箭头线型混淆；
- 图例缺项、移位、翻译或增加第五项；
- 出现台词、轴线、道具、衔接、镜头功能或段落说明；
- 生图模型生成了乱码、伪文字、镜号、时码或图例；
- 在 `PRECISION-GRID` 中用整页生图冒充逐镜资产；
- 在 `ROUGH-GRID` 中无必要地逐镜生成、拖慢交付或展示中间单镜；
- 为填满五格而复制、合并、删除镜头或改变既定镜号顺序。

## 约束

- 拼版不得重绘、拉伸或覆盖源图片。
- 默认 `contain` 完整保留构图；明确允许裁切时使用 `--fit cover`。
- 人物方向用实线箭头，摄影机方向用虚线箭头。
- 中文标题、镜号和时码由程序绘制，避免生图模型生成错误文字。
- 缺图允许生成灰色占位页，但正式交付前必须消除全部缺图警告。

## 每格必备信息

正式宫格每格至少显示：

```text
稳定镜号：S01-B01-C01（同时作为唯一显示镜号）
成片时码：起点—终点
镜头功能：关系建立 / 反应 / 动作 / 揭示等
决定性画面：人物、动作相位、关键道具和空间关系
景别/机位：全景、中景、近景、角度与观察位置
人物运动：方向与终点；静止时写“静止”
摄影机运动：类型、方向、速度与终点；固定时写锁定对象
台词/声音：DIA / OS / VO / AMB / SFX / MUS / SIL 摘要
衔接：同向接 / 反向接 / 静切动 / 动切静 / 动作接动作
连续性：轴线、视线、持物手、道具和状态要点
```

当版面空间有限时，画面外最低保留：`镜号 + 时码 + 功能 + 运镜 + 声音/台词摘要 + 衔接`。完整稳定镜号和连续性可进入页脚或随附 manifest，但不得丢失。

## 箭头语法

- 人物运动：黑色实线箭头；起点靠近人物，箭头指向动作终点。
- 摄影机运动：黑色虚线箭头；不得与人物实线箭头混淆。
- 推近/拉远：使用朝向画面中心或远离画面中心的虚线箭头，并在文字区标注“推/拉”。
- 摇摄/俯仰：使用弧形虚线箭头，并写明最终锁定对象。
- 固定镜头：不画摄影机箭头，文字标注“固定｜锁定对象”。
- 焦点转移：不作为摄影机位移箭头；用 `焦点 A → B` 单独标注。
- 任何箭头必须有可命名终点；没有终点的运动不得进入正式宫格。

## 方向、轴线与衔接

以下情况必须在 manifest 和宫格中记录人物方向与摄影机方向：人物走动、进出门、转身、双人对话、多人调度、动作场、狭长空间和跨页延续动作。

相邻格统一使用五类衔接：

1. `同向接`：维持人物或摄影机能量方向；
2. `反向接`：用于揭示、逆转或情绪回收；
3. `静切动`：事件启动；
4. `动切静`：钉住结果、证据或停顿；
5. `动作接动作`：同一人物动作或物件运动跨格延续。

双人对话必须保持屏幕方向与视线匹配。需要换轴时，用中性镜头、沿轴镜头、可见越轴运动或新关系镜重新建立，不能只靠箭头掩盖空间错误。

## 黑白草图验收

每张草图在进入拼版前检查：

- 第一眼是否能分辨人物身份和主动作；
- 左右位置、视线和身体朝向是否符合空间圣经；
- 是否选中决定性姿态，而非模糊过渡帧；
- 景别变化是否与分镜母版一致；
- 关键道具是否出现且状态正确；
- 同一场景的门、桌、窗、光源和人物比例是否连续；
- 是否意外新增人物、道具、文字或剧情；
- 是否留有足够安全区供程序叠加箭头。

任一项失败时只重画相关格，不重做已成立的整页。

## 禁止交付

- 只有画面、没有镜号和时码；
- 只有箭头、没有方向终点与文字解释；
- 用生图模型生成镜号、中文台词或精确时码；
- 为快速拼版拉伸、重绘或覆盖原始单镜图；
- 用漂亮写实图替代空间、动作和剪辑信息；
- 将缺图占位页作为正式完成品。
- 将整页多格生图、无标注多格草图或近似版式当作已确认样式交付。

# 专业剧本转分镜

把文学剧本转译为“能拍、能剪、能生成、能复核”的镜头母版。先保护故事事实，再做导演设计；先建立空间和表演，再决定摄影机；先完成镜头逻辑，再适配具体生成平台。

## 总控接入

由 `aigc-production-orchestrator` 调用时，本 Skill 是 S3 阶段编排器：以 `P01@locked`、`S01@locked`、`A01@locked` 和所需 A02–A05 为权威输入，统一产出 `B01` 镜头母版、`B02` 连续性账本，以及按需生成的 `B03` 宫格故事版。已有任务契约和材料不重复询问；故事或资产问题登记后分别回 S1/S2，不在分镜里静默修正。只有 B01/B02 通过交付闸口后才能向 S4 移交。

## 总原则

1. 保留原剧情、台词属性、事件顺序、人物关系和因果。除非用户明确授权改编，不补戏、不改词、不提前后续事件。
2. 每镜必须有可命名的叙事功能。写不出功能时删除或合并。
3. 只写摄影机能拍到、麦克风能录到或后期能实现的事实；将心理词转成动作、表演、构图、光声或道具证据。
4. 先规划人物调度和空间轴线，再规划机位与运镜。运镜必须由信息、情绪、动作或视点变化驱动。
5. 同时检查故事连续性、空间连续性、动作连续性、造型/道具连续性和声音连续性。
6. 将“成片时长”与“素材生成时长”分开。短促剪辑镜可以由更长、更稳定的素材截取。
7. 15 秒仅在平台、用户或交付链明确要求时作为硬容器；否则按自然戏剧节拍划分 sequence/beat。
8. 任何越轴、跳切、失焦、时间跳跃或风格突变都必须有明确意图，并提供观众重新定位的办法。
9. 输出以制作可执行性为准，不堆砌焦段、光圈和导演名词。未知条件标记为假设，不伪造确定性。
10. 不把“稳定”误写成“全部固定”。人物位移、空间揭示、危险逼近、关系变化或动作爆发明显时，主动设计场景运动弧线和至少一个有记忆点的运动节点。
11. 不把“美感”简化为三分法、对称或浅景深。每镜必须有第一视觉点、第二视觉点、注意力路径和明确的构图功能；背景、留白、光色与几何都要参与叙事。
12. 不把“对白需要多久”直接等同于“单镜持续多久”。先保护自然口语总时长，再按语义变化、动作相位、视线和听话人反应拆 coverage；超长单镜必须由内部调度、构图重组、焦点交接或明确长镜意图支撑。
13. 不把固定镜头作为安全默认值。固定镜头用于关系锚定、反应、停顿和结果标点；信息推进、动作变化、关系变化或空间揭示时，优先设计有动机、有方向、有终点的运动。
14. 不把快节奏等同于镜头多或镜头短。节奏取决于单位时间内有效状态变化、未完成牵引和信息重排；观众已经读懂的画面立即切出，不为完整展示动作尾巴、礼貌反应或空余波停留。

## 选择工作模式

根据用户材料自动选择，缺少非关键参数时用默认值继续，不连续追问。

### 原文权限

- `LOCKED`：默认。只做视听转译和执行细化。
- `DIRECTOR`：允许重排镜头表达，不改变剧情事实和台词。
- `ADAPT`：仅用户明确要求改编时启用；把改写与分镜分区输出。

### 制作轨道

- `LIVE`：真人/实拍，强调 coverage、调度、镜头可拍性和剪辑余量。
- `ANIMATION`：动画/预演，强调 pose、silhouette、staging、镜头节奏。
- `AI`：AI 视频，额外控制单镜复杂度、资产绑定、状态跳变和失败降级。
- `HYBRID`：默认；先给专业镜头母版，再附 AI 执行字段。

### 节奏风格

- `RESTRAINED`：克制叙事，允许长镜、留白和画外空间。
- `CLASSIC`：默认，连续性剪辑与戏剧节拍优先。
- `COMMERCIAL`：商业短剧/竖屏，信息密度和钩子优先，但不牺牲可读性。
- `COMMERCIAL_FAST`：商业快节奏；晚进早出、限制 coverage、提高有效变化密度，以未完成动作/声音牵引下一镜。
- `ACTION`：动作场，空间因果与打击反馈优先。

用户明确说“节奏慢、加快、紧凑、少停顿、高信息密度、不要完整反应”时自动选择 `COMMERCIAL_FAST`；普通商业短剧但未要求提速时选择 `COMMERCIAL`。不要仅因画幅是 9:16 就强制快切。

## 标准工作流

### 1. 建立任务契约

在内部确认并在输出头部简记：

- 原文权限、制作轨道、节奏风格、画幅、目标时长/平台；
- 节奏目标：普通/快速/高压快切、预计平均镜长、固定组内关键变化数；
- 交付粒度：仅节拍、分镜表、宫格图规划、AI 母版或连续性审计；
- 可用资产：角色、场景、道具、参考图；
- 明确限制：固定组时长、镜头数、审核尺度、预算或设备。

画幅未指定时沿用原项目；短剧可合理假设 9:16，电影/剧集可合理假设 16:9，但必须在输出头部注明假设。

### 2. 原文保护扫描

建立故事事实账本：

- 场次、时间、内外景、人物、道具；
- 事件顺序、因果、信息揭示顺序；
- 台词、OS、VO、旁白的属性；
- 进入状态和离开状态。

发现逻辑缺口时列入“剧本问题”，继续处理不受影响的部分。除非获得 `ADAPT` 权限，不自行修补。

### 3. 拆戏剧节拍

以“局势发生可感知变化”为切点，不以段落或固定秒数机械切分。每个 beat 标记：

- 人物目标与阻力；
- 信息变化或权力变化；
- 情绪起点 → 转折 → 落点；
- 观众此刻必须看见/听见什么；
- 最适合的组尾或转场载体。

为每个关键 beat 做双视角对账：演员此刻在做什么、压抑什么、身体哪里泄露；观众此刻应先看见什么、误读什么、最后理解什么。两者不对齐时，调整表演、景别、构图或补必要镜头。

建立轻量锚点：`戏剧动作｜情绪起点→过渡→落点｜视觉锚点｜声音锚点｜表演锚点`。不强制固定情境 ID，也不把镜头组合锁死。

不要把每次眨眼、停顿、转头或语气变化都升级为新 beat。微表演只有改变人物选择、权力关系、观众理解或动作结果时才独立成 beat；否则留在镜头内部。`COMMERCIAL_FAST` 下，每个 15 秒段优先包含至少 2 次有意义的局势/信息变化，并保留一个未完成因果、动作或声音牵引下一段；不靠重复反应、空镜或完整动作尾巴填满时长。

### 4. 建立空间圣经与场面调度

在正式拆镜前锁定：入口、出口、人物初始位、关键道具、主动作轴线、屏幕方向、光源方向和可用机位区。多人或动作场必须输出简洁站位说明。

先写人物走位、视线和身体朝向，再写摄影机。若人物调度变化导致轴线改变，给出中性镜头、沿轴镜头、可见过线运动或新的建立镜头。

### 5. 设计 coverage 与镜头弧线

先选择场景总体策略，再拆单镜：

- 对话：建立关系镜 → 主体/反应 coverage → 必要插入 → 权力变化后的新关系镜；
- 动作：地理建立 → 意图 → 动作 → 接触/结果 → 反应 → 新局势；
- 情绪：克制证据 → 裂缝 → 决定性细节 → 余波；
- 悬疑：受限信息 → 线索 → 误读/延迟 → 揭示或保留；
- 长镜：写清每次走位、焦点转移、构图重组和退出方案。

每镜先写功能，再写画面、表演、声音、摄影机和剪辑点。镜头大小、角度、焦段与运动应构成场景弧线，而不是逐镜随机变化。

`COMMERCIAL_FAST` 使用以下切分纪律：

- 晚进：从动作、冲突或信息已经启动的位置进入，删除观众能够推断的准备、走近、坐下、寒暄和重复建立。
- 早出：信息、结果或情绪一旦可读立即切走，不等待完整收势、礼貌反应或静态余波结束。
- 合并建立：优先在人物进入、动作启动或线索显现的同一镜中完成空间/关系建立，不额外支付纯建立镜。
- 限制反应：反应只有改变下一步选择、权力、信息或情绪方向时才独立成镜；否则放在双人镜、画外声 coverage 或同镜内部。
- 牵引切点：优先在未完成动作、未答问题、视线目标、声音先行、遮挡或运动方向上切，让下一镜接管能量。
- 功能去重：连续两镜若主功能相同且没有新增节奏贡献，合并或删除；构图不同不是保留理由。

长对白先划分语义单元，再设计视觉 coverage。允许台词跨镜连续，并用 OS、L-cut 或 J-cut 把画面交给听话人反应、关系镜、手势或关键道具；不得为了切镜加快口语，也不得为了保留完整口型让缺少内部变化的单一构图长时间占据成片。竖屏商业短剧推荐单镜 2-6 秒；7 秒允许但必须复核；8 秒及以上必须拆镜。长对白、焦点变化、轻微表演或轻微运镜均不得作为 8 秒及以上的自动豁免理由。只有用户明确指定长镜头，并在该镜写入 `长镜授权：USER-REQUESTED`，才允许保留 8 秒及以上单镜。其他类型仍按场景判断，不机械套用此硬阈值。

按职责把镜头标为：

- `MASTER` 主镜：承担 beat 的核心信息、动作或情绪；
- `SUPPORT` 辅镜：补空间、反应、物件、表演或剪辑连续性；
- `ACCENT` 标点镜：以短促细节或声画命中改变节奏；
- `TRANSITION` 转场镜：连接时间、空间或视点。

每个 beat 至少一个 `MASTER`。删除后不损失信息、动作、情绪、空间或节奏的 `SUPPORT/ACCENT` 必须删除。

`COMMERCIAL_FAST` 下，普通 beat 默认镜头预算为 `1 MASTER + 最多 1 SUPPORT`；仅当动作相位、空间重定位或一次明确标点无法同镜表达时增加 1 个 `ACCENT`。超过 3 镜必须在 Beat 卡写明 `超预算理由`。不得为每句台词机械配置说话人、听话人、手部和关系镜；优先让台词通过 OS、L-cut 或 J-cut 跨镜持续。

按 [composition-and-visual-design.md](references/composition-and-visual-design.md) 先定场景视觉圣经，再定单镜：构图母题、几何关系、主要明暗/色彩分区、景深策略、负空间方向和视觉锚点。每镜写清第一视觉点、第二视觉点、视线动线、前中后景和构图如何在镜头终点发生变化。

同时按 [camera-movement-grammar.md](references/camera-movement-grammar.md) 设计场景运动句法：

- 先选择运动强度 `M0-M4` 和主导运动语言；
- 再为每个 beat 指定“静止锚点、发展运动、强调运动、收束运动”中的职责；
- 需要强画面感时，优先使用“一个平台位移 + 一个取景/焦点修正”的可读组合；
- 写清起点、路径、速度曲线、终点、主体关系和下镜承接；
- 让前景、中景、远景产生视差、遮挡揭示或尺度变化，而不只让摄影机机械靠近。

人物接近危险、进入新空间、追逐、坠落、跃迁或关系逆转时，默认至少达到 `M2`；高潮可使用一个 `M3` 关键节点。只有美学或叙事明确要求时才全段保持 `M0/M1`。

逐场检查固定镜头密度：连续两个固定镜头必须承担不同信息；连续三个固定镜头必须说明节奏目的，否则至少将其中一镜改为由动作或信息驱动的 M1/M2。增加运动时不得机械地“每镜微推”；必须写清运动改变了什么信息、人物距离或构图关系，并在明确终点停住。

### 6. 核算时长与对白容量

按实际语言估算对白时长，并为呼吸、反应、动作和剪辑留空间。不得通过擅改台词解决超载。

分别核算“台词连续时长”和“单镜成片时长”。一段对白可以自然持续 10-15 秒，但应由多个有功能的镜头共同承载。竖屏商业短剧不得因为对白连续而让单镜达到 8 秒；必须拆成说话人、听话人、关系、动作或道具 coverage。只有用户明确指定长镜头并写入授权字段时，才可突破阈值。

`COMMERCIAL_FAST` 的建议成片范围：普通对白/信息镜 `1.5-3.5s`，动作或标点镜 `0.8-2.5s`，关键情绪命中 `3-5s`。这些是诊断区间，不是机械切镜器；单镜达到 `6s` 必须填写 `节奏豁免`，说明内部至少两次有效变化或不可替代的悬念保持。逐场核算平均镜长与中位镜长，同时检查连续慢镜、SUPPORT 占比和重复镜头功能。

当固定组时长不足时，按顺序处理：移动自然切点 → 跨组延续 → 拆分动作阶段 → 报告台词超载。只有用户授权时才压缩或改写。

### 7. 写正式镜头母版

使用 [storyboard-master-template.md](assets/storyboard-master-template.md) 或 [output-schema.md](references/output-schema.md) 的字段。镜号必须唯一且稳定：`S01-B02-C03`。

每镜至少包含：

- 成片时码/时长、镜头功能、节奏贡献与必要的节奏豁免；
- 镜头职责、第一/第二视觉点、构图策略、视线动线、层次与光色分区；
- 主体、可见动作、人物表演；
- 景别、角度、构图、机位/焦段；
- 运镜强度、组合、起点、路径、速度曲线、终点、主体关系及动机；
- 台词/声音；
- 剪辑点与承接；
- 连续性状态；
- AI 轨道下的风险、素材时长和降级方案。

一镜只承担一个主要摄影机任务。快切组要拆成可独立生产的子镜，不把多个素材藏在一个镜号里。

### 8. 做五层连续性审计

逐个切点检查：

1. 故事：信息、因果、时间是否顺序正确；
2. 空间：轴线、屏幕方向、视线、相对位置是否可读；
3. 动作：动作阶段、入画/出画方向、运动向量是否承接；
4. 状态：服装、发型、伤痕、持物手、道具位置、光线是否一致；
5. 声音：台词属性、环境底、声桥、音乐和静默是否连续。

对每个问题标记 `ERROR`、`WARN` 或 `INTENTIONAL`。`INTENTIONAL` 必须写叙事理由和重定位手段。

### 9. 做 AI 风险拆解

仅 `AI/HYBRID` 执行。按 [ai-video-production.md](references/ai-video-production.md) 评估单镜复杂度。优先降低并发变化，不降低戏剧功能：

1. 固定或简化摄影机；
2. 减少同镜多人复杂动作；
3. 把动作拆成起势、接触、结果；
4. 用插入、遮挡、反应或声音替代不稳定直拍；
5. 延长素材生成时长，后期截取；
6. 最后才建议改变叙事方案，并明确说明影响。

### 10. 通过交付闸口

交付前确认：

- 原文事实和台词属性无未经授权的变化；
- 每个 beat 有镜头承载，每镜有功能；
- 时长总计与目标一致，固定组满足硬时长；
- 镜号唯一、时码递增、无隐藏素材；
- 高风险切点已有修复或明确意图；
- 角色、空间、道具和声音状态可跨镜追踪；
- AI 高风险镜头有降级方案；
- 输出能直接交给画师、摄影、剪辑、动画或视频生成环节。
- 竖屏商业短剧不存在未经用户明确授权的 8 秒及以上单镜；7 秒镜头已复核；
- 连续固定镜头已有明确节奏目的，场景运动弧线既不是无动机乱动，也不是全部静止；
- `COMMERCIAL_FAST` 每镜均有可验证的节奏贡献；普通 beat 遵守镜头预算，超预算已有理由；
- `COMMERCIAL_FAST` 不存在连续两镜同功能且无新增信息、连续三镜均达到 4 秒以上、或用重复反应/动作尾巴填时长；

逐镜执行价值四问：

1. 本镜服务哪个 beat 和哪个微观瞬间？
2. 删除本镜是否损失信息、动作、情绪、空间或节奏？
3. 构图、表演、声音和运动是否存在更清楚、更有力的表达？
4. 第一眼看哪里、随后看哪里、为何在此切镜是否明确？

任一问题无法回答时重做、合并或删除该镜。

将正式分镜保存为 Markdown 时，运行：

```bash
python3 scripts/validate_storyboard.py path/to/storyboard.md
```

修复所有 `ERROR` 后再交付；`WARN` 可以保留，但要向用户说明。

## 输出层级

根据用户要求选择最小充分交付：

- `OUTLINE`：场次 + 节拍 + 预计镜头数；
- `BOARD`：完整专业镜头母版；
- `GRID`：在 BOARD 基础上给宫格页、格号和画面关键帧规划；
- `AI-PACK`：在 BOARD 基础上加资产绑定、生成时长、风险和降级；
- `AUDIT`：不重写分镜，只列问题、证据、严重度和修复建议。

## 宫格故事版拼版

当用户要求“宫格图、故事版、storyboard sheet、把分镜图拼版”时，在 BOARD 之后执行确定性拼版，不用生图模型重绘整张宫格：

### 宫格格式硬锁定

- 用户已经确认某种宫格样式后，后续“继续、重新生成、继续绘制、下一页”等指令一律继承该样式；除非用户明确要求换版式，不得退回通用宫格、详细导演卡片或无标注草图页。
- `black-white-director-5` 必须严格使用保存的样式配置和视觉样板。不得凭文字描述近似复刻，也不得自行改变格数、画布比例、标题位置、图例位置、边框、留白或箭头语法。
- 每格标头必须直接使用 BOARD 中的稳定专业镜号 `Sxx-Bxx-Cxx`，例如 `S01-B01-C01  0.0–3.0s`。禁止另建全片递增的 `C01、C02、C03……` 显示号；进入新 Beat 后严格沿用 BOARD 镜号，从该 Beat 的 `C01` 开始。
- 根据用途选择两种生产模式：黑白草图、快速参考、审镜和用户要求“直接出宫格”时默认 `ROUGH-GRID`；角色资产级、正式拍摄执行或用户明确要求逐镜精修时使用 `PRECISION-GRID`。
- `ROUGH-GRID`：允许生图模型一次生成整页五格无文字画面，随后由确定性程序添加镜号、时码、箭头和图例；只调用一次画面生成，不逐镜生成，不向用户展示中间素材。
- `PRECISION-GRID`：逐镜生成无文字画面，再确定性拼版。不得把该高成本流程误用为黑白快速参考的默认方案。
- 禁止把“尚未添加镜号、时码、箭头和图例的多格草图”称为宫格成品或正式预览。
- 无论使用哪种模式，精确文字与箭头都必须由程序叠加。若程序尚不支持选定 preset，先补齐/修复程序，不得让生图模型乱写标注。
- 正式输出前必须与 `assets/black-white-director-grid-style-reference.png` 做版式核对。任何一项不符即判定失败并返工：上三下二、右下图例、格顶镜号时码、实线人物箭头、虚线摄影机箭头、无额外说明文字。

1. `ROUGH-GRID` 生成整页无文字画面源；`PRECISION-GRID` 将单镜图片用完整镜号命名并绑定；
2. 用 `scripts/parse_storyboard_md.py` 生成 manifest；
3. 按画幅选择 `vertical-6`、`classic-6` 或 `director-5`；
4. 用 `scripts/compose_storyboard_grid.py` 输出逐页 PNG 和合并 PDF；
5. 用 `scripts/validate_storyboard_grid.py` 检查缺图、重复镜号和输出完整性。

正式交付不得保留缺图占位；人物运动使用实线箭头，摄影机运动使用虚线箭头。详细流程见 [storyboard-grid-layout.md](references/storyboard-grid-layout.md)。

纯黑白草图只算宫格图的画面层，不算完整 GRID 交付。每格必须补齐镜号、时码、镜头功能、人物/摄影机方向与终点、台词或声音摘要、衔接和连续性要点。不得让生图模型生成这些精确文字；统一由拼版脚本从 BOARD/manifest 绘制。移动人物必须记录屏幕方向，移动摄影机必须记录方向、终点和衔接类型；焦点转移不得误画成摄影机位移箭头。

例外：用户明确要求“只参考示例标注、其他文字都不要”时，使用 `black-white-director-5` 极简输出样式：每格仅保留 `S01-B01-C01  0.0–3.0s`，画内使用实线人物箭头、虚线摄影机箭头、弧移/俯仰箭头，右下角放四项英文图例；不得添加轴线、台词、道具、衔接或说明段落。样式配置与视觉样板见 `assets/black-white-director-grid-style.*`。

用户未指定时默认输出 `BOARD`；篇幅过长时先交付场次总览和首场完整分镜，再分批继续，保持镜号和状态账本连续。

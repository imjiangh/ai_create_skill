## 1. 阶段与状态

固定阶段顺序：

`S1_SCRIPT → S2_ASSETS → S3_STORYBOARD → S4_VIDEO_PROMPTS`

阶段状态只允许：

- `not_started`：尚未开始。
- `in_progress`：当前唯一执行阶段。
- `review`：工件已产出，等待闸门或必要确认。
- `locked`：闸门通过，可作为下游权威输入。
- `needs_revision`：因本阶段问题或上游变更失效。
- `blocked`：缺少外部必需信息或权限，无法安全继续。
- `not_applicable`：项目明确不需要该阶段或工件。

工件状态允许 `missing`、`draft`、`review`、`locked`、`needs_revision`、`not_applicable`。

## 2. 标准工件

| ID | 工件 | 所有者 | 最低内容 |
|---|---|---|---|
| P01 | 项目档案 | S1 | 类型、平台、受众、时长、画幅、主题、人物、世界观、风格、权限 |
| S01 | 制作锁定剧本 | S1 | 场次、动作、台词属性、角色关系、关键道具/VFX、版本 |
| S02 | 剧本诊断与修订记录 | S1 | 问题、证据、决定、修订范围、遗留风险 |
| A01 | 资产总表 | S2 | 人物/场景/道具/VFX ID、名称、首次出现、状态、引用 |
| A02 | 人物资产包 | S2 | 身份锁、比例、服装、造型、视图、版本；人物图不混入道具 |
| A03 | 场景资产包 | S2 | 空间结构、时代、材质、光源、入口出口、尺度、版本 |
| A04 | 道具资产包 | S2 | 外观、尺寸、归属、左右手/佩挂、状态变化、连续性 |
| A05 | VFX 资产包 | S2 | 系统定义、组件、状态、强度、交互、合成和连续性；无 VFX 时 N/A |
| B01 | 镜头母版 | S3 | 镜号、时码、叙事功能、画面、调度、摄影机、表演、声音、衔接 |
| B02 | 连续性账本 | S3 | 人物/道具/空间/动作/光色/声音的入镜和出镜状态 |
| B03 | 宫格故事版 | S3 | 决定性画面、镜号时码、运动标注；非必需时可 N/A |
| V01 | 项目级视频风格锁 | S4 | 唯一主风格、画幅、帧率/质感、光色、运动和禁止漂移项 |
| V02 | 视频 Prompt 包 | S4 | 按大分镜组独立可粘贴，逐镜引用 B01 镜号与资产 ID |
| V03 | 视频生成清单 | S4 | Prompt 文件、参考图槽、时长、模型/参数、版本、依赖和状态 |

## 3. 阶段闸门

### G1 剧本锁定

- P01 与 S01 均为 `locked`。
- 故事因果、人物目标和关键事件不存在 P0/P1 未决项。
- 每场有明确功能；目标时长和格式可执行。
- 台词属性、旁白/OS/VO、关键道具与 VFX 已标明。
- 原文权限为 `LOCKED`、`DIRECTOR` 或 `ADAPT`，且修改没有越权。

### G2 资产锁定

- A01 为 `locked`，每个资产都具有稳定 ID。
- A02–A05 为 `locked` 或 `not_applicable`。
- 人物身份、服装与身体比例跨图一致；人物资产不携带独立道具。
- 场景的入口、出口、尺度、主光方向可供分镜使用。
- 道具归属、左右关系和状态变化可追踪。
- VFX 的系统、状态和交互明确，不是泛化形容词。

### G3 分镜锁定

- B01 与 B02 为 `locked`；B03 可锁定或 N/A。
- 所有剧本事实都有镜头承载，没有偷偷改戏。
- 时长、对白容量、轴线、视线、动作和声音连续性通过。
- 镜头引用的资产 ID 均存在且版本匹配。
- 每镜具备叙事功能、注意力路径、运动动机和可生成性。

### G4 视频 Prompt 锁定

- V01、V02、V03 均为 `locked`。
- 每个 Prompt 可独立投喂，并引用明确的 B01 镜号和资产图槽。
- 构图、动作、表演、光影、摄影机、内部运动、声音和转场可解析。
- 不复述或改写已由资产图锁定的外观，不发明新剧情。
- Prompt 时长总和、镜头顺序和 B01 一致。

## 4. 交接包

阶段交接必须包含：

```text
from_stage: Sx
to_stage: Sy
authority: LOCKED | DIRECTOR | ADAPT
inputs_used: [工件ID@版本]
outputs_locked: [工件ID@版本 + 路径]
decisions: [不可被下游重解释的决定]
assumptions: [仍在使用的假设]
open_issues: [问题ID + 严重度 + 所有者]
accepted_risks: [用户已接受风险]
next_action: 唯一推荐动作
```

## 5. 问题所有权与返工

- 故事事实、人物动机、结构节奏、台词内容：S1。
- 人物外观、场景结构、道具状态、VFX 系统：S2。
- 机位、轴线、调度、镜头时长、衔接、覆盖：S3。
- 单段 Prompt 的解析性、动作密度、模型约束、图槽声明：S4。

下游只能登记并路由上游问题，不能静默修正。P0/P1 回到所有者阶段；P2 可附说明继续。

## 6. 变更影响矩阵

| 变更 | 默认失效范围 |
|---|---|
| P01 类型/平台/画幅/时长/主风格 | S1–S4 重新审计 |
| S01 情节、场次、台词、角色、关键动作 | 相关 A01 条目、B01/B02、V02/V03 |
| A02 人物身份或服装 | 引用该人物的 B01/B03、V02/V03 |
| A03 场景结构或光源 | 引用该场景的 B01/B02/B03、V01/V02/V03 |
| A04 道具归属/状态 | 相关 B01/B02/B03、V02/V03 |
| A05 VFX 系统/状态 | 相关 B01/B02/B03、V02/V03 |
| B01 镜序、时长、构图、动作 | 对应 V02/V03 |
| B02 连续性 | 对应 B01/B03 与 V02/V03 |
| V01 风格锁 | 全部 V02/V03 |

只失效受影响的范围；状态账本记录影响说明。

## 7. 命名与版本

- 项目 ID：短英文或拼音，例如 `FY01`。
- 工件版本：`<ID>-v<major>.<minor>`，例如 `B01-v1.2`。
- 不兼容的故事、身份、空间或输出契约变化提升 major；局部修正提升 minor。
- 镜号跨版本保持稳定；删除镜头保留墓碑记录，不复用旧镜号。
- 文件名建议：`<项目ID>_<工件ID>_<简短名称>_v<版本>.<ext>`。
- 状态账本只引用真实路径；不把聊天中的临时文本标成 `locked`。

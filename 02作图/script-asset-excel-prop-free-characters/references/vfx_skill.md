# 影视 VFX 视觉系统资产生成 Skill V1.0

## 0. Skill 定位

本 Skill 用于把剧本中的“特效描述”转化为一套可以被导演、美术、VFX、后期、AI 作图和图生视频共同使用的 **VFX 视觉系统资产**。

本 Skill 不把 VFX 当成一张漂亮概念图，而是把每项 VFX 定义为一个完整系统：

- 它是什么
- 为什么出现
- 从哪里产生
- 作用于谁或什么
- 具体长什么样
- 按什么规律运动
- 何时出现、达到峰值、衰减和消失
- 如何影响人物、道具、环境和镜头
- 哪些部分必须固定
- 哪些部分只能锁定规律、不能锁死轮廓
- 如何在不同集数、场景和镜头中保持一致
- 如何生成资产图、状态图、拆解图和视频执行提示词

本 Skill 可独立运行，不依赖其他 Skill 或外部 Schema。

# 1. 核心原则

## 1.1 VFX 必须被定义为“系统”

禁止只输出：

> 紫色能量环绕人物，电影感，华丽粒子，强烈发光。

必须至少定义：

- 发射源
- 作用对象
- 空间范围
- 轮廓与材质
- 主运动
- 次运动
- 生命周期
- 颜色与发光层次
- 环境交互
- 镜头与合成要求
- 变体与连续性规则

## 1.2 分离“外观”和“运动”

每个特效必须分别定义：

### 外观层 Appearance

- 形态
- 材质
- 密度
- 透明度
- 纹理
- 主色
- 高亮色
- 边缘色
- 发光结构
- 局部细节
- 稳定识别特征

### 动态层 Motion

- 发射方向
- 主运动轨迹
- 次级扰动
- 速度
- 加速度
- 脉冲频率
- 聚散规律
- 旋转方向
- 重力或浮力倾向
- 是否附着主体
- 是否受风、水流、冲击或人物动作影响
- 消散方式

外观可以固定，动态必须按物理和叙事规律变化。

## 1.3 分离“固定图形资产”和“动态模拟系统”

以下内容通常需要锁定固定图形：

- 阵法
- 符文
- 封印纹
- 图腾
- 邪术纹号
- 经络纹
- 护心纹
- 法器发光纹路
- 星图连线
- 节点拓扑
- 机械或科技界面
- 可识别的伤痕、裂纹和感染纹理

以下内容通常不能锁死单帧轮廓，只能锁定视觉规律：

- 烟
- 雾
- 火焰
- 气流
- 水流
- 黑水蔓延
- 粒子
- 火星
- 灵气
- 毒雾
- 能量丝
- 魂体碎屑
- 爆炸
- 破碎云
- 环境扬尘

## 1.4 先可信，再华丽

每项 VFX 必须拥有可理解的现实参考或运动逻辑，例如：

- 烟流涡旋
- 水中墨汁
- 毛细扩散
- 电晕放电
- 磁场线
- 音波共振图形
- 玻璃破裂
- 灰烬崩解
- 菌丝生长
- 油膜干涉
- 水母生物光
- 流沙、灰尘、飞絮
- 植物根系、血管、闪电分叉

虚构特效可以超现实，但不能无规律乱动。

## 1.5 必须包含完整生命周期

所有动态 VFX 默认使用六阶段：

1. 潜伏 Latent
2. 触发 Trigger
3. 生成 Formation
4. 增长 Growth
5. 峰值 Peak
6. 衰减与残留 Decay / Residue

若剧情只出现其中部分阶段，也必须说明缺失阶段的原因。

## 1.6 必须考虑镜头融合

任何进入真实镜头的 VFX，都必须定义：

- 前后遮挡
- 与人物或物体的接触边缘
- 环境投光
- 接触阴影
- 反射
- 折射
- 景深
- 运动模糊
- 空气透视
- 镜头曝光
- 暗部可读性
- 是否影响头发、衣物、水面、灰尘或环境雾

禁止输出“贴纸式特效”。

## 1.7 连续性高于单张美观

同一 VFX 系统跨镜头必须保持：

- 主色
- 核心轮廓逻辑
- 发射源
- 运动方向
- 速度等级
- 粒子密度
- 发光层次
- 状态等级
- 残留结果
- 与角色或法器的绑定关系

# 3. 默认输出配置

若用户没有指定，采用以下默认值：

- 资产图画幅：16:9 横版
- 视觉定位：真实电影 VFX 视觉开发
- 细节风格：大结构优先，细节克制，避免廉价粒子堆砌
- 光影：单一明确主光方向，暗部保留信息
- 资产背景：
  - 固定纹样和图形资产：中性深灰或纯黑背景
  - 半透明体积与发光资产：中性深灰背景
  - 镜头融合示意：使用剧情场景背景
  - 材质与残留资产：中性实物测试背景
- 中文提示词：完整中文
- Midjourney 提示词：纯英文
- MJ 参数：`--ar 16:9 --style raw --s 100`
- 视频提示词：中文导演执行格式
- 特效强度：克制、可信、可拍摄、可合成
- 禁止方向：手游技能特效、廉价仙侠光效、无意义粒子风暴、霓虹灯管感、卡通魔法贴纸

用户指定画幅、模型、项目风格或交付格式时，以用户要求为最高优先级。

# 4. 总体工作流

严格按以下顺序执行。

## 阶段 A：读取与提取

1. 阅读全部输入。
2. 按集、场、镜头识别所有 VFX 实例。
3. 不仅识别明确写出的“特效”，也识别隐性 VFX：
   - 异常光
   - 异常雾气
   - 环境侵蚀
   - 超自然痕迹
   - 物体自主运动
   - 灵魂显形
   - 能量传递
   - 身体异变
   - 阵法激活
   - 远程探查
   - 瞬移、隐身、时间停滞
   - 水、火、风、雷、冰、尘、碎片等异常运动
4. 记录首次出现、重复次数、作用对象和状态变化。

## 阶段 B：合并为 VFX 系统

同一视觉逻辑的多个镜头合并成一个 VFX 系统。

例如：

- “胸口黑纹”
- “蚀纹扩散”
- “蚀纹受压制”
- “蚀纹反噬”

应合并为：

> 双重蚀纹视觉系统

但不同视觉机制不得强行合并。

例如：

- 人体上的双重蚀纹
- 空气中的黑蛇蚀气

虽然剧情相关，仍应拆为两个系统，再记录二者关系。

## 阶段 C：类型判断

为每个系统指定一个主类型和零到三个辅类型。

主类型必须从第 5 节分类库中选择。

## 阶段 D：资产边界判断

判断该系统属于：

- 固定图形资产
- 动态视觉系统
- 角色绑定特效
- 道具绑定特效
- 场景绑定特效
- 镜头专属特效
- 环境残留资产
- 复合 VFX 系统

## 阶段 E：建立视觉圣经

按第 8 节完整填写 VFX 系统卡。

## 阶段 F：生成资产图方案

按系统需求选择第 10 节中的资产图类型，不得机械地让每个系统都生成同一组图片。

## 阶段 G：生成提示词

输出：

- 中文主视觉资产提示词
- 中文状态演化图提示词
- 中文组件拆解图提示词
- 中文镜头融合提示词
- 固定图形资产提示词
- 纯英文 Midjourney 提示词
- 视频动态执行提示词
- 负向提示词

## 阶段 H：连续性和执行校验

按第 16 节进行逐项验证，修正后再交付。

# 5. VFX 分类库

## V01 体积与气态类 Volume / Pyro

适用：

- 烟
- 雾
- 云
- 黑气
- 灵雾
- 火焰
- 爆炸烟云
- 蒸汽
- 毒气
- 怨气
- 体积能量

必须定义：

- 发射源
- 密度
- 温度感
- 浮力倾向
- 涡旋尺度
- 扩散范围
- 边缘软硬
- 是否遮挡主体
- 是否吸光或发光

## V02 液体与流体类 Liquid / Fluid

- 重力
- 附着性
- 渗透性
- 飞溅尺度
- 湿润和残留

复合系统必须先拆为子层，再定义统一触发关系。

# 6. 资产级别判断

每个 VFX 系统必须标记一个主要资产级别。

## A 级：固定核心资产

必须严格锁定：

- 图形
- 拓扑
- 节点
- 比例
- 方向
- 标识
- 纹理结构

适合：

- 阵法
- 符文
- 蚀纹
- 封印
- 法器纹路
- 科技界面

## B 级：可复用动态系统

锁定运动规律，不锁死每一帧轮廓。

适合：

- 黑气
- 灵华
- 火焰
- 雾
- 黑水
- 粒子
- 电弧

## C 级：角色或道具绑定系统

必须定义锚点和附着关系。

适合：

- 心脉纹
- 手掌能量
- 武器发光
- 身体侵蚀
- 灵力灌注

## D 级：场景绑定系统

必须定义场景材质和空间范围。

适合：

- 废井怨气
- 墙面黑水
- 地面阵法
- 房间结界
- 水面异动

## E 级：镜头专属系统

高度依赖特定构图、机位或动作，不能作为独立通用资产使用。

适合：

- 角色被能量击中
- 灵魂从身体中抽离
- 阵法包围多人
- 远程探查视野
- 传送门中的人物穿越

## F 级：环境残留资产

可作为材质或贴花重复使用。

适合：

- 灼烧痕迹
- 腐蚀痕迹
- 黑水痕迹
- 冰霜
- 灰烬
- 裂纹
- 灵力残光

# 7. VFX 系统合并与拆分规则

## 应合并

满足以下多数条件时合并：

- 同一发射源
- 同一视觉机制
- 同一主色和材质
- 同一运动规律
- 同一剧情因果
- 仅强度或状态不同

## 应拆分

满足任一关键条件时拆分：

- 固定纹样与动态气流同时存在
- 发射源不同
- 运动机制不同
- 作用对象不同
- 一个可以独立复用
- 一个属于环境残留
- 一个是角色绑定，一个是场景绑定
- 一个需要固定拓扑，一个不能锁死形状

## 系统关系

拆分后必须记录关系：

- 前置触发
- 同步激活
- 强弱联动
- 压制关系
- 转化关系
- 崩解关系
- 残留关系

# 8. VFX 系统卡 Schema

每个系统必须完整填写以下字段。

## 8.1 基础信息

- VFX 系统 ID
- VFX 系统名称
- 别名
- 主类型
- 辅类型
- 资产级别
- 首次出现
- 出现频次
- 关联角色
- 关联道具
- 关联场景
- 关联 VFX 系统

## 8.2 剧情功能

- 剧情用途
- 观众应理解的信息
- 情绪目标
- 视觉优先级
- 是否为全剧核心视觉线索
- 是否承担推理、身份、危险或能力等级信息

## 8.3 触发与作用

# 9. 生命周期标准

## S0 潜伏

特效未完全显形，只能通过微弱异常感知。

常见表现：

- 极淡纹理
- 局部冷暖变化
- 微弱折射
- 偶发粒子
- 低频脉冲
- 局部材质改变

## S1 触发

特效从无到有的明确起点。

必须定义：

- 首个亮点或裂点
- 首个运动方向
- 首个声音提示
- 触发延迟
- 是否伴随人物动作

## S2 生成

结构开始形成。

必须定义：

- 生成顺序
- 由点、线、面还是体积开始
- 是否沿人体、物体或地面生长
- 是否存在扫描、勾勒、凝聚或喷发

## S3 增长

体量、亮度、范围或速度增加。

必须定义：

- 增长曲线
- 扩散方向
- 是否出现次级层
- 是否压迫环境
- 是否开始影响人物

## S4 峰值

特效最强状态。

必须定义：

- 最大范围
- 最大亮度
- 最大密度
- 最强交互
- 是否产生冲击、破坏或转化
- 视觉中心位置

## S5 衰减

能量下降或结构瓦解。

必须定义：

- 从哪里开始消退
- 是否向内塌缩
- 是否向外碎裂
- 发光先消失还是体积先消失
- 是否出现余辉、烟尘、液体或碎屑

## S6 残留

特效结束后对环境或主体留下的可追踪结果。

常见表现：

- 焦痕
- 水痕
- 裂纹
- 灰烬
- 结霜
- 湿润
- 发光余痕
- 纹样残片
- 衣物和皮肤状态变化

# 10. VFX 资产图类型

不得只生成一张“最终状态图”。按系统需要选择以下类型。

## T01 主视觉定义图

目的：

- 锁定整体视觉
- 锁定主色、材质、体量和识别特征

画面要求：

- 16:9 横版
- 单一主体
- 中性背景
- 不使用复杂叙事构图
- 大结构优先
- 特效边缘、透明度和内部层次可读
- 不堆砌无意义粒子

## T02 状态演化图

固定推荐为 1×6 横版：

1. 潜伏
2. 触发
3. 生成
4. 增长
5. 峰值
6. 衰减与残留

同一主体、同一机位、同一比例、同一光线。

## T03 组件拆解图

分区展示：

- 核心主体
- 体积层
- 丝线或电弧层
- 粒子层
- 发光层
- 折射层
- 接触层
- 残留层

要求每个组件可独立识别，不混成一团。

## T04 空间与锚点图

用于角色、道具或场景绑定特效。

展示：

- 发射源
- 锚点
- 作用区域
- 运动方向
- 最大范围
- 与主体比例
- 禁止越界区域

## T05 镜头融合示意图

用于说明：

- 特效在人物前后关系
- 遮挡
- 投光
- 反射
- 接触阴影
- 空气透视
- 景深
- 运动模糊
- 环境响应

这张图可以使用剧情背景，但只用于执行，不作为固定纯资产。

## T06 固定图形正视资产图

适用于：

- 阵法
- 符文
- 蚀纹
- 封印
- 星图
- 科技图形

要求：

- 严格正视
- 无透视
- 图形完整
- 节点、线条和比例可读
- 中性纯色背景
- 可用于贴图、投影、合成或二次描绘

## T07 固定图形状态图

至少展示：

- 未激活
- 局部激活
- 完全激活
- 过载
- 断裂
- 熄灭残留

图形拓扑不得漂移。

## T08 材质与残留资产图

适用于：

- 腐蚀
- 焦痕
- 黑水
- 裂纹
- 冰霜
- 灰烬
- 发光残痕

要求展示：

- 干净材质
- 侵蚀后的材质
- 边缘过渡
- 强弱三档
- 近景细节
- 大面积铺展状态

## T09 尺度对照图

适用于难以判断体量的 VFX。

展示：

- 人体比例
- 房间比例
- 道具比例
- 最大范围
- 最小可读范围

## T10 镜头专属关键帧

用于 E 级资产。

必须绑定：

- 镜头编号
- 景别
- 机位
- 主体动作
- 特效状态
- 时间节点

# 11. 参考研究方法

对每项 VFX，至少提供三类参考。

## 11.1 现实物理参考

例如：

- 烟流
- 火焰
- 液体
- 折射
- 电弧
- 破碎
- 冲击波
- 流沙
- 毛细扩散
- 薄膜干涉

## 11.2 生物或自然参考

例如：

- 蛇
- 鳗鱼
- 水母
- 菌丝
- 根系
- 血管
- 闪电
- 极光
- 鱼群
- 虫群
- 花粉
- 灰烬

## 11.3 影视语义参考

只描述可借鉴的机制，不复制受版权保护的具体设计。

必须写成：

- 参考其体积密度
- 参考其慢速盘旋逻辑
- 参考其核心高亮与外围衰减
- 参考其环境投光
- 参考其低频压迫感

禁止写成：

> 完全照搬某影片某角色的特效。

# 16. 校验规则

交付前必须逐项检查。

## 16.1 系统识别

- 是否漏掉隐性 VFX
- 是否把同一系统重复拆成多个名称
- 是否把不同机制错误合并
- 是否记录首次出现和频次
- 是否建立相关系统关系

## 16.2 固定与动态边界

- 固定纹样是否有正视拓扑资产
- 动态烟雾是否被错误锁死为固定轮廓
- 角色绑定特效是否有锚点
- 环境残留是否被单独记录
- 镜头专属效果是否错误当成通用资产

## 16.3 生命周期

- 是否存在触发点
- 是否说明生成顺序
- 是否说明峰值
- 是否说明衰减
- 是否有残留
- 视频提示词是否含时间节点

## 16.4 外观与动态

- 外观和动态是否分开定义
- 是否有现实参考
- 是否有主运动和次运动
- 是否有速度等级
- 是否有空间范围
- 是否出现无规律随机运动

## 16.5 颜色与发光

- 是否定义核心高亮、主体色、边缘色和环境染色
- 是否避免整团均匀发光
- 是否考虑暗场和亮场
- 是否与其他 VFX 区分
- 是否禁止颜色漂移

## 16.6 合成

- 是否有正确遮挡
- 是否有人物和环境投光
- 是否有接触阴影
- 是否考虑景深与运动模糊
- 是否考虑反射、折射或热扰动
- 是否避免遮挡关键表演

## 16.7 提示词

- 中文提示词是否完整
- MJ 提示词是否纯英文
- 是否保留画幅参数
- 是否包含系统专属负向提示
- 是否避免空泛词堆叠
- 是否避免只写“cinematic magical glowing particles”
- 是否没有水印、文字和 UI

## 16.8 连续性

- 同一系统颜色是否一致
- 发射源是否一致
- 锚点是否一致
- 纹样是否一致
- 运动方向是否一致
- 镜头状态是否正确继承
- 残留是否在后续镜头保留

# 19. 强度等级

同一系统默认建立四档强度。

## L0 潜伏

仅能被仔细观察发现。

## L1 可见

观众能明确看见，但不压过人物表演。

## L2 强化

承担剧情重点，明显影响环境和人物。

## L3 峰值

用于高潮或失控，体量、亮度、速度或破坏达到最高。

强度变化不能改变系统身份。

例如紫色灵华从 L1 到 L3 可以增加：

- 亮度
- 范围
- 粒子
- 脉冲
- 环境染色

但不能突然变成蓝色闪电或金色火焰。

# 20. 常见错误与自动修正规则

## 错误 1：特效只有形容词

错误：

> 神秘、华丽、震撼、电影感。

修正：

补充发射源、运动、材质、生命周期和环境交互。

## 错误 2：所有奇幻特效都是发光粒子

修正：

优先判断体积、液体、纹样、屏障、材质变形和残留，不得默认使用粒子。

## 错误 3：烟雾轮廓每镜完全相同

修正：

锁定密度、颜色、运动方向、附着关系和尺度，允许自然流动变化。

## 错误 4：阵法每镜都不同

修正：

建立固定正视图、节点图和状态图，镜头中仅允许透视和亮度变化。

## 错误 5：发光不影响人物和环境

修正：

添加皮肤、服装、墙面和地面的环境染色，以及正确接触阴影和反射。

## 错误 6：特效突然出现又突然消失

修正：

增加触发、生成、衰减和残留。

## 错误 7：人物动作与特效无关

修正：

把发射源和锚点绑定到具体身体部位、道具或空间位置，并定义动作反馈。

## 错误 8：视频提示词没有时间结构

修正：

强制加入秒级时间段和阶段变化。

## 错误 9：多种 VFX 颜色混淆

修正：

建立项目级颜色体系，为不同力量、阵营和状态分配独立核心色、主体色和边缘色。

## 错误 10：过度追求细节

修正：

优先锁定三组大结构：核心主体、外围体积、交互与残留；细节只作为辅助。

# 21. 项目级 VFX 视觉体系

当一个项目包含多个 VFX 系统时，先建立项目级规则。

必须定义：

- 正向力量色彩体系
- 负向力量色彩体系
- 中性或未知力量色彩体系
- 体积特效共同材质
- 纹样共同语法
- 粒子尺度范围
- 发光强度上限
- 暗场可读规则
- 环境投光规则
- 禁止的廉价视觉方向
- 不同系统之间的区分方式

例如：

```text
正向灵元：紫白核心、深紫主体、淡青边缘
负向蚀气：无纯黑死块，以墨黑、油灰、暗紫边缘构成
封印系统：闭合几何、向内收束
攻击系统：方向性强、速度快、拖尾明确
防护系统：闭合、稳定、受击时产生局部波纹
魂体系统：保留人体结构，低密度体积与边缘碎散
```

# 22. 示例：蚀气与黑蛇缠绕视觉系统

## 基础判断

- 主类型：V01 体积与气态类
- 辅类型：V04 发光与能量类、V08 感染与异变类
- 资产级别：B 级可复用动态系统 + C 级角色绑定系统
- 关联固定资产：双重蚀纹
- 不应锁死：每一缕黑气的具体轮廓
- 必须锁定：
  - 墨黑、油灰、暗紫边缘
  - 贴近头顶、胸口和命脉盘旋
  - 蛇状不是实体蛇，不出现清晰蛇头、眼睛和鳞片
  - 慢速盘旋与突然收紧并存
  - 受灵元压制时向胸口内塌缩
  - 不向整个房间无限扩散

## 推荐资产图

- T01 主视觉定义图
- T02 状态演化图
- T03 组件拆解图
- T04 人体锚点图
- T05 镜头融合示意图

## 负向方向

- 禁止实体黑蛇
- 禁止怪物触手
- 禁止浓黑死块
- 禁止紫色霓虹灯
- 禁止火焰运动
- 禁止全身均匀包裹
- 禁止无源粒子
- 禁止遮挡角色面部
- 禁止离开身体独立飞行

## 中文主视觉资产提示词示例

```text
【画面核心】
蚀气与黑蛇缠绕视觉系统影视 VFX 主视觉定义资产图，16:9 横版，用于锁定黑色蚀气的体积、颜色、附着位置、蛇状运动暗示和压迫感，不是实体蛇，不是剧情海报。

【剧情与视觉功能】
该特效表现负向侵蚀力量长期盘踞人物头顶、胸口与命脉，观众需要感受到隐蔽、黏附、压制与随时可能收紧的危险。

【VFX类型与资产级别】
主类型为体积与气态类，辅类型为能量类与感染类，属于可复用动态系统和角色绑定系统。

【发射源与空间关系】
蚀气从胸口双重蚀纹和命脉附近缓慢渗出，以胸骨中心、颈侧和头顶上方为主要锚点，贴近人体轮廓盘旋，最大范围不超过人物身体外侧约三十厘米，不向房间无限扩散。

【核心外观】
主体由低密度油烟、冷雾和水中墨汁般的半透明体积混合构成，整体墨黑与油灰，不形成纯黑死块；局部气流拉成长而收束的蛇状流线，但绝不出现清晰蛇头、眼睛、鳞片或实体身体。边缘带极淡暗紫衰减，内部存在缓慢卷动的密度层。

【颜色与发光】
核心不主动强发光，以墨黑和油灰吸收环境光，局部边缘出现极淡暗紫冷光，用于区分轮廓；禁止紫色霓虹灯感和整团均匀发亮。

【动态暗示】
通过贴身弧形流线、局部拖尾和胸口附近的内向旋涡，暗示其主运动为缓慢盘旋和黏附，次运动为低频收紧与轻微脉冲。受压制时所有流线朝胸口内塌缩。

【组件层次】
清楚区分胸口渗出的核心密度层、贴身盘旋的体积层、蛇状细流层、极少量暗紫微粒层和贴近皮肤衣物的接触阴影层。

【构图与镜头】
使用中性灰人体测试模型或克制的人形轮廓作为尺度参考，主体居中，正面三分之二身，背景中性深灰，保留干净留白，清楚显示头顶、颈侧、胸口和命脉位置关系。

【制作质感】
真实电影级 VFX 视觉开发，物理可信，潮湿、黏附、压迫，大结构优先，细节克制，暗部可读。

【禁止】
```

# 24. 最终交付标准

一个合格的 VFX 系统必须能够回答：

- 导演知道它在剧情里为什么出现
- 美术知道它的形态和材质
- VFX 知道它应拆成哪些层
- 动画或视频模型知道它如何运动
- 合成知道它怎样进入镜头
- 制片知道它是通用资产还是镜头专属
- 后续镜头知道如何继承状态
- AI 作图知道该生成哪种资产图
- AI 视频知道时间节点和动态变化
- 所有人都知道哪些东西绝对不能变

无法回答以上问题的 VFX 描述，不得作为最终资产定义交付。

# 25. VFX 输出图片固定模板（浅色清晰版）

本节用于统一所有 VFX 资产图的输出样式。若用户没有另行指定，则默认所有 VFX 资产图优先使用本模板。

## 25.1 模板目标

此模板的目标不是做气氛海报，而是：

- 在浅色背景上清楚看见特效本体
- 保持轮廓、透明度、体积、发光和边缘结构可读
- 便于导演、美术、VFX、AI 作图和图生视频共同识别
- 便于后续做连续性比对
- 避免深色背景把细节吃掉
- 避免纯白背景导致发光边缘和半透明结构丢失

因此，默认背景不是纯黑，也不是死白，而是 **浅色中性展示背景**。

## 25.2 默认展示背景

除非用户特别指定，默认采用：

- 背景主色：浅冷灰、浅暖灰或极浅灰蓝
- 背景明度：高明度，但不刺眼
- 推荐质感：干净、均匀、轻微渐变
- 禁止：纯黑背景、脏污纹理背景、复杂叙事背景、强烈彩色背景、过亮死白背景

推荐背景逻辑：

### 标准背景 A：浅冷灰展示底

适用于大多数 VFX 系统。

- 主背景：浅冷灰
- 局部渐变：中心略亮、四周略暗
- 作用：让暗色、烟雾、半透明、紫光、黑水、蚀气都清楚可见

### 标准背景 B：浅暖灰展示底

适用于金色、铜色、火焰类、灵光类、法阵类资产。

- 主背景：浅暖灰
- 局部渐变：中心干净、边缘轻微压暗
- 作用：避免金光和暖色边缘融进背景

### 标准背景 C：极浅灰蓝展示底

适用于冷色灵体、魂魄、屏障、寒气、结界和透明膜类特效。

- 主背景：极浅灰蓝
- 作用：让冷色透明结构更清晰，同时不失电影质感

## 25.3 统一画面结构

默认所有单张 VFX 资产图采用以下固定版式：

### 单图版式

- 画幅：16:9 横版
- 构图：主体居中但不顶满画面
- 留白：四周留出足够安全边距
- 尺度：特效本体占画面宽度约 45%–70%
- 地面：可有极弱接地阴影或淡接触影，用于判断空间关系
- 禁止：复杂道具陈列、杂乱前景、剧情性人物表演、海报式大透视

### 状态板版式

- 1×6 横版状态演化板
- 每格同机位、同背景、同主体尺度、同光线
- 每格之间用干净留白分隔
- 不加花哨边框
- 若必须标注阶段，只能使用极轻量小字，且不喧宾夺主

### 拆解图版式

- 左侧或上方为完整效果
- 右侧或下方分区展示组件层
- 每个组件拥有独立展示区域
- 组件之间保持统一背景和同一色彩体系
- 不允许多个组件重叠成一团看不清

## 25.4 浅色模板的核心光影规则

因为背景变浅，必须控制以下问题：

### 轮廓优先

- 特效边缘必须清楚
- 半透明体积必须能看见厚薄变化
- 发光边缘和实体边缘要分层
- 黑色或深色特效不得融进背景

### 发光克制

- 发光只在核心区域和关键边缘增强
- 禁止整团纯白过曝
- 禁止光晕无限扩散把细节洗掉
- 发光层要保留色相，不得全部烧成白光

### 体积可读

- 烟、雾、灵气、魂体等必须保留内层密度差
- 浅色背景上，体积结构要靠明暗层次和边缘密度区分，而不是只靠一圈光

### 轻投影与轻接触影

- 若主体需要空间感，可以在底部加入极轻的接地阴影
- 若为附着在人物或道具上的特效，应有极轻接触阴影或接触暗边
- 阴影不得抢画面，不得把展示图做成剧情光效图

## 25.5 默认制作风格描述

所有浅色模板默认追加以下风格要求：

- 轮廓清楚
- 大结构优先
- 细节克制
- 发光不过曝
- 半透明结构可读
- 材质统一干净
- 轻微柔焦边缘

- 通透、高级、可执行

## 25.6 各类资产图的浅色模板规范

### T01 主视觉定义图（浅色版）

- 背景：标准背景 A 为默认
- 主体：单一 VFX 系统
- 目标：一眼看清轮廓、材质、透明度、颜色层级和体量
- 必须：主体与背景形成足够明度反差
- 禁止：黑底炫光海报风、深色氛围图、过度彩烟

### T02 状态演化图（浅色版）

- 背景：统一浅冷灰或浅暖灰
- 六格亮度一致
- 不允许每格换背景色
- 特效强弱变化依靠系统本身，不靠背景变黑或变亮制造假对比

### T03 组件拆解图（浅色版）

- 背景：统一浅灰
- 每层单独清楚展示
- 建议完整效果使用稍深一档浅灰底，组件层使用更干净的浅灰底
- 目的：清楚看出每层职责

### T04 空间与锚点图（浅色版）

- 背景：浅灰
- 人体、道具或场景轮廓应尽量简化、克制
- 重点是锚点、作用范围和最大边界清楚
- 辅助主体不应喧宾夺主

### T05 镜头融合示意图

此类允许使用真实剧情背景，不强制浅色底。但若用户要求统一展示版，也可额外提供一张浅色中性示意版，用于说明遮挡、投光和接触关系。

### T06 固定图形正视资产图（浅色版）

- 背景：极浅灰或浅灰蓝
- 图形必须完整、正视、无透视
- 线条边缘清晰
- 发光图形须控制亮度，保证节点和线条不糊成白团

### T07 固定图形状态图（浅色版）

- 背景统一
- 图形拓扑一致
- 状态差异通过发光强度、局部激活范围、断裂和残留体现
- 禁止靠背景变化表现激活程度

### T08 材质与残留资产图（浅色版）

- 背景：浅灰实物测试底
- 建议加入简化材质块或平面测试板
- 便于清楚看见腐蚀、焦痕、黑水、裂纹、霜痕等边缘过渡

### T09 尺度对照图（浅色版）

- 背景：浅灰
- 参考对象简洁清楚
- 尺度线和范围清晰但克制
- 重点是大小关系，不是做科幻UI图表

### T10 镜头专属关键帧

按剧情需要执行，不强制浅色底；但若用于方案汇报，可同步补一版浅色展示关键帧。

# 26. 固定信息板模板锁定（参考图同款）

当用户提供类似“Crimson Shard Serpent”这类浅色信息板，并明确表示“我想用这套模板”时，默认启用本节，作为 VFX 资产图的固定展示模版。

本节优先级高于第 25 节的一般浅色模板描述。

## 26.1 模板定位

这套模板不是普通概念图，也不是剧情海报，而是：

> **影视 VFX 视觉系统信息板 / Asset Presentation Board / Look Development Board**

它同时承担：

- 主视觉定义
- 状态演化
- 组件拆解
- 材质近景
- 色彩与透明度规范
- 关键信息摘要

因此，后续所有核心 VFX 资产图，优先采用“**单张信息板式输出**”，而不是只给一张单独特效图。

## 26.2 固定版式结构

默认画幅：**16:9 横版**。

整张信息板固定分为六个区块：

### A 区：左上标题与摘要区

位置：左上纵向信息栏。

固定内容：

- VFX 系统名称（大标题）
- 副标题 / 一句话定义
- 关键词标签（3–6 个）
- 一句视觉标语或描述句
- 基础属性摘要：
  - element / 类型
  - behavior / 行为
  - trigger / 触发方式
  - motion / 运动特征
  - signature trait / 标志特征

中文项目中可以改为：

- 系统名称
- 系统一句话定义
- 关键词
- 核心视觉短句
- 特效类型
- 运动行为
- 触发条件
- 运动方向
- 标志特征

要求：信息简洁、结构整齐、不要写成大段散文。

### B 区：中央主视觉展示区

位置：画面中央最大区域。

固定内容：

- 该 VFX 的完整主视觉形态
- 最清晰、最有代表性的系统状态
- 优先展示整体轮廓、动势、体积、密度、主色、发光和空间关系

要求：

- 中央主视觉必须是整张板的视觉中心
- 主体占据全板最主要面积
- 不要被文字遮挡
- 不要同时塞入多种不相关状态
- 尽量让人一眼看懂“这个 VFX 长什么样”

### C 区：右上状态演化区

位置：右侧纵向栏。

固定内容：

- 状态演化 / State Progression
- 推荐 5 格或 6 格
- 每格展示一个连续状态
- 每格包含：
  - 状态编号
  - 状态名称
  - 一句简述
  - 缩略视觉图

推荐顺序：

1. 潜伏 / 初始聚集
2. 展开 / 激活
3. 完整形态 / 峰值
4. 释放 / 冲击
5. 衰减 / 消散

如果项目更复杂，也可使用六阶段：

1. 潜伏
2. 触发
3. 生成
4. 增长
5. 峰值
6. 衰减与残留

要求：

- 所有小图必须同风格、同尺度逻辑
- 不得变成五种不同特效
- 每个状态都能看出与中央主视觉的关系

### D 区：左下组件拆解区

位置：左下区域。

固定内容：

- Layer Breakdown / 组件分层拆解
- 用清晰条目展示该系统由哪些视觉层组成
- 每层配：
  - 小色点 / 小图标
  - 组件名称
  - 一句定义
  - 对应单独视觉层示意

建议层包括：

- Core / 核心主体层
- Inner Reflection / 内层反射层
- Metallic Highlight / 高亮层
- Spark / 火花或微粒层
- Debris / 外散碎屑层
- Motion Gap / 透明空隙层
- Directional Trails / 方向拖尾层

实际命名按 VFX 类型调整，但必须具备“分层可读性”。

### E 区：中下细节近景区

位置：底部中央区域。

固定内容：

- Detail Close-ups / 细节近景
- 建议 4–6 格
- 每格展示局部放大细节
- 每格下方配一个极短名称

适合展示：

- 边缘形态
- 材质结构
- 发光细节
- 反射层
- 粒子密度
- 裂纹或破损方式
- 透明叠层
- 残留纹理

要求：

- 每格都必须真的是“有用细节”，而不是重复主图裁切
- 细节图要帮助后续制作，不是装饰

### F 区：右下色彩与透明度规范区

位置：右下区域。

固定内容：

- Color & Opacity Palette
- 5–7 个颜色样本方块
- 每个方块标注：
  - 颜色名称
  - 16 进制色值（可选）
  - 用途
  - 建议透明度范围
- 底部可加入透明度或强度渐变条

适用信息：

- 核心高亮色
- 主体色
- 反射色
- 边缘衰减色
- 环境染色色
- 阴影色
- 低透明度余辉色

要求：

- 色板必须服务于执行
- 不允许只是好看但无用途的装饰色卡

## 26.3 整体风格锁定

使用该模板时，所有信息板默认锁定以下风格：

- 浅米白、浅暖灰或极浅灰底

- 轻排版感

- 图像是主角，排版是辅助

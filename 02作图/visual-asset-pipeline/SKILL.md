# 视觉资产编排

作为 S2 阶段唯一编排器。执行顺序固定为“先清单和 ID，后提示词与图，最后一致性锁定”。禁止边生成边随意改命名。

## 控制模式

由总控调用时，继承 P01、S01、权限、版本和硬锁。按需读取叶子 Skill；叶子输出回到本 Skill 汇总，不单独宣告项目完成。

## 能力路由

1. 资产提取、四表 Excel、人物无道具规则、道具/VFX 分离：读取 [`script-asset-excel-prop-free-characters`](../script-asset-excel-prop-free-characters/SKILL.md)。涉及 VFX 时按其要求完整读取 VFX 权威参考。
2. 人物图像提示、生成、检查或修复：读取 [`generate-pristine-people`](../generate-pristine-people/SKILL.md)。
3. 场景图像提示、生成、检查或修复：读取 [`generate-pristine-environments`](../generate-pristine-environments/SKILL.md)。
4. 实体道具沿用 A01/A04 的定义生成独立资产提示；不得把可拿取、交换、丢失或使用的物体混入人物资产图。
5. 无对应生成器的资产类型，仍先形成可执行资产定义和提示词，再由总控选择合适生成能力。

## 输入契约

最低输入：

- `P01@locked`：项目格式、风格、画幅和角色/世界观约束。
- `S01@locked`：制作锁定剧本。
- 剧情硬锁与任何已有参考图。

已有资产可接管，但必须先归入 A01、补 ID、来源、版本和可信状态。

## 内部流程

### 1. 全量提取

逐场扫描 S01，提取人物、场景、实体道具和 VFX 系统。记录首次出现、出现场次、剧情用途、状态变化和预计复用频率。

### 2. 分类与去重

- 同一资产的状态变化优先作为同一 ID 的 variant，不随意新建 ID。
- 可独立拿取、交换、丢失、使用或追踪连续性的物体归 A04。
- 非实体的能量、烟火、液体、粒子、异变、魂体、阵法和复合效果归 A05。
- 人物资产只保留身体、妆发、服装结构和不可拆造型，不携带 A04。

### 3. 建立资产总表

先完成 A01，再开始生成。命名建议：`C###` 人物、`E###` 场景、`P###` 道具、`F###` VFX；项目已有命名体系时沿用但保持唯一。

每条至少包含：ID、标准名、别名、类别、来源场次、剧情功能、状态/变体、提示词状态、图像状态、版本、下游引用。

### 4. 资产圣经与提示词

从 P01/S01 提取不可漂移字段。人物先锁身份和比例，场景先锁几何和光源，道具先锁尺寸/归属/左右关系，VFX 先锁系统与生命周期，再写平台提示词。

### 5. 生成与条件修复

按资产优先级生成：核心人物 → 核心场景 → 剧情关键道具 → 核心 VFX → 次要资产。每次检查只针对当前资产类型；局部失败优先定向修复，不无理由全量重抽。

### 6. 跨资产一致性审计

检查人物与服装、人物与道具比例、场景与人物尺度、光源、时代材质、VFX 交互、左右手和状态演化。冲突回到其资产定义，不在分镜阶段补救。

### 7. 锁定

锁定 A01，并将 A02–A05 标为 `locked` 或 `not_applicable`。未生成图片但已有可执行定义时只能标 `draft/review`，不得假装锁定。

## S2 闸门

- A01 中没有重复 ID、幽灵资产或未分类的剧情关键对象。
- A02 人物身份和比例一致，且无独立道具混入。
- A03 能支持入口、出口、站位、轴线、尺度和主光方向。
- A04 能追踪归属、左右关系与每个关键状态。
- A05 能追踪系统、组件、强度、生命周期、交互和合成逻辑。
- 所有分镜必需资产均锁定；非必需缺口已说明。

## 交接输出

```text
A01：资产总表路径 + 版本 + locked
A02–A05：路径/目录 + 版本 + locked 或 not_applicable
资产硬锁：身份、服装、空间、光源、道具状态、VFX 状态
引用规则：资产 ID → 文件/图槽
分镜限制：尺度、入口出口、左右关系、禁止组合
遗留 P2：问题 + 可接受范围
```

S3/S4 发现资产不一致时，本 Skill 只修对应资产及受影响引用，并通知总控失效范围。

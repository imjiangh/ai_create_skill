# 剧本开发编排

作为 S1 阶段唯一编排器。不要把五份专家文件当成五个并列聊天机器人；根据问题类型按需读取并调用，统一向总控返回 `P01`、`S01` 和 `S02`。

## 控制模式

由 `aigc-production-orchestrator` 调用时：

- 使用总控已经提供的材料、权限、版本和目标，不重复索取。
- 跳过叶子文件的开场自检、身份宣告、再次确认和会话结束话术。
- 保留叶子文件的专业流程、硬规则、诊断维度、输出字段和禁忌。
- 叶子专家只返回本次分配的结果；本 Skill 决定下一位专家和返工去向。

独立使用时，也按本文件统一编排，不要求用户手动切换五个对话。

## 专家路由

| 任务/问题 | 读取文件 | 职责 | 不负责 |
|---|---|---|---|
| 灵感、定位、项目档案、大纲 | [`01-剧本创作.md`](../01-剧本创作.md) | 从零建档和定方向 | 完整场景剧本 |
| 按档案和大纲写稿/返稿 | [`02_0-剧本写作.md`](../02_0-剧本写作.md) | 写完整剧本和执行修订 | 独立质量裁决 |
| 钩子、三幕、场次功能、时长、主角能动性 | [`02_1-剧本节奏专家.md`](../02_1-剧本节奏专家.md) | 结构诊断与重构方案 | 台词逐句精修 |
| 多视角盲点、市场与制作会诊 | [`03-剧本医生.md`](../03-剧本医生.md) | 只诊断并归纳证据 | 直接改稿 |
| 逐句台词、潜台词、角色语言指纹 | [`04-台词专家.md`](../04-台词专家.md) | 台词诊断和局部改写 | 改剧情结构 |

遇到复合问题时先结构、后场次、再台词。不要用台词润色遮盖结构问题。

## 输入契约

接受任意成熟度的输入；内部归一为：

- `authority`：`LOCKED` / `DIRECTOR` / `ADAPT`。
- `project_brief`：平台、类型、受众、时长、画幅、主题、风格。
- `source_material`：想法、梗概、大纲、剧本或既有版本。
- `revision_request`：需要解决的问题与不可改变项。
- `downstream_constraints`：资产、分镜、生成平台已经锁定的限制。

缺少非关键参数时做显式假设继续；只有会改变故事方向、人物身份或交付规模时才请求确认。

## 默认生产流程

### 1. 建档与定纲

没有可靠 P01/大纲时读取 `01-剧本创作.md`，完成创意收敛、项目档案和大纲。已有合格档案时只补缺口，不重跑七步。

### 2. 形成剧本 v1

读取 `02_0-剧本写作.md`。按项目类型选择模式、节奏和分集开关，产出完整可审剧本。超长项目分批，但保持场次、集数和版本连续。

### 3. 结构节奏闸门

读取 `02_1-剧本节奏专家.md`，至少检查：钩子、三幕/段落结构、场次功能、剧情增量、冲突升级、主角能动性、信息释放、集尾动力、目标时长。

发现 P0/P1 时，形成明确的删/并/移/增清单，交回剧本写作专家返稿；不要直接进入台词精修。

### 4. 独立会诊

结构初步通过后读取 `03-剧本医生.md`。会诊只产出证据、分歧和主治归纳，不在会诊角色内改稿。

把归纳拆为问题项：

- 结构/因果/人物行动：回剧本写作或节奏专家。
- 台词：进入台词专家。
- 拍摄/资产风险但不改故事：记录给 S2/S3。
- 纯偏好且无证据：不自动触发返稿。

### 5. 返稿与复检

让 `02_0` 执行已确认的修改单；必要时由 `02_1` 复检受影响部分。默认最多两轮完整循环；仍有 P0/P1 时报告真实分歧，不无限润色。

### 6. 台词专检

结构和场次锁定后读取 `04-台词专家.md`。只改台词层；若一句台词必须靠改剧情才能成立，登记为上游问题，不越权处理。

### 7. 制作锁定

统一术语、角色名、场次号、OS/VO/旁白属性、关键道具、VFX、状态变化和目标时长。生成 S02 修订记录并锁定 S01。

## S1 闸门

只有以下条件全部满足才能返回 `PASS`：

- P01 与 S01 版本明确且内容一致。
- 没有未决 P0/P1 故事问题。
- 每场有功能，人物目标与因果连续。
- 台词属性和关键动作可被镜头化。
- 人物、场景、道具、VFX 候选可被 S2 提取。
- 用户明确要求保留的内容没有被静默改写。

## 交接输出

```text
P01：项目档案路径 + 版本 + locked
S01：制作锁定剧本路径 + 版本 + locked
S02：诊断与修订记录路径 + 版本
剧情硬锁：不可被下游改变的事实
资产候选：人物 / 场景 / 道具 / VFX
分镜注意：时长、复杂调度、特殊声音、生成风险
遗留 P2：问题 + 接受理由
```

下游只能提出问题，不能直接改 S01。收到返工时按问题所有权读取最少必要的专家文件，修复受影响范围并重新过闸。

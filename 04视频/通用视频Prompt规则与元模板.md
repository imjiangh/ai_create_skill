# 文生/图生视频 Prompt 通用规则与元模板
### （面向 Seedance 类视频生成模型 · 可直接交付给任意 LLM 作为写 prompt 的系统规则）

> **总控接入协议**：独立粘贴使用时按全文执行；由 `video-prompt-pipeline` 或 `aigc-production-orchestrator` 调用时，本文件是 S4 的 Prompt 写作规则库。以传入的 V01、A01–A05、B01、B02 和镜号范围为准，只产出被分配的 Prompt 单元及审计结果；不得改写剧情、重新设计资产、重排镜序或自行宣布项目完成。发现上游冲突时返回问题及其所有者。

> 本文档是一套与具体项目无关的通用方法论 + 元模板。把它整份交给一个 LLM，它就能按统一标准为你写出高质量、低抽卡成本、画面可控的视频 prompt。核心目标只有一个：**让同一条 prompt 生成 10 次，画面差别都不大**，把抽卡成本压到最低。

---

## 第一部分：十三条核心原理（写每一条 prompt 都必须遵守）

### 原理 1 ·【降抽卡成本】构图必须钉死到"抽10次不跑偏"
这是第一性原则。模型的自由发挥空间 = 你的抽卡成本。prompt 留白越多，每次生成差异越大，越费钱费时。每个镜头的构图必须写满四要素：
- **位置**：主体在画面的哪里（画面左侧三分之一 / 正中偏下 / 前景底部偏左 / 中部……）
- **占比**：主体占画面多少（约二分之一画幅 / 占画面五分之四 / 占三分之一成绝对主体……）
- **景别 + 前中后景**：是特写/近景/中景/全景/大远景；前景、中景、远景分别是什么
- **朝向**：主体面朝哪（背对镜头 / 四分之三侧朝右 / 正面 / 仰头……）

自检：写完每个镜头问"这句让模型抽10次会不会出10个样？"——会，就是没钉死，继续加锚。

### 原理 2 ·【细节饱满，但不外包给模型】
绝不写"音效要饱满""表情要丰富""动作要自然""画面要有张力"这类形容词——这是把活儿层层外包给模型自己悟，模型只能解析具象事实，悟不出来。要"饱满/丰富/痛苦/虚脱"，你就**自己拆成可解析的层**：
- 一声干呕"饱满" → 喉头闷顿 → 气流撕过声带嘶拉 → 腹腔挤压低沉腔体回响 → 浊气喷出顿挫闷响（由低到高再砸落）
- 表情"痛苦" → 眼睛眯起又艰难睁开 + 眉头死拧 + 嘴角向下抽搐 + 半边脸扭曲
- 体态"虚脱" → 手指虚弱抓挠 → 手臂颤抖撑起 → 脑袋沉垂 → 动作后手臂一软脱力瘫倒
形容词是你偷懒的信号；拆成动词和可见事实，才是真的写清楚。

### 原理 3 ·【翻译而非挪用】
用户/编剧口语里的比喻、玩梗、IP 参照、结构意图，绝不能原样搬进 prompt，必须先翻译成模型能解析的具象画面：
- **比喻拆成视觉事实**："烤串似的" → 尖物从中央穿透、戳挂在顶端；"猫鼠扭打成团" → 扭成贴地翻滚的尘土团球、四肢支棱出来、看不清里面是谁。
- **玩梗避开会让模型跑偏的字眼**："蜻蜓点水" → 又快又轻的连环点踢，并避开"点水/潮水"等会让模型跑出水的字。
- **IP 参照只取视觉本质、不写 IP 名**："像某游戏角色骑小马" → 只取"夸张体型差的荒诞喜感"，不写 IP 名（既防版权污染、也防模型乱抓 IP 视觉）。
- **多动作"同步"要明写揉在一起**："他摇晃着干呕然后倒下"若是同时发生，要写"晃悠、干呕、栽倒揉在一起一气呵成"，不能拆成顺叙的三个独立步骤，否则模型会一个一个分开演、很怪。
- 自检信号：发现自己在照抄原话，就是该翻译的信号。

### 原理 4 ·【强调谁就给谁画面】
嘴上说"它很大/它很重要"对模型无效。要强调体型/重要性，必须落到画面占比和尺度对照：
- 体型大 → "占画面五分之四、如一堵肉墙、躯长约三四个成年人首尾相接"
- 体型悬殊 → **三级体型阶梯**钉死：最大的如移动山丘（占大半画高）→ 中等的仅及它一条腿高 → 最小的小如蚂蚁。用一条可见的尺度链把悬殊焊死，否则模型会把所有东西归一化成差不多大。

### 原理 5 ·【正向锚定，禁反向】
多数视频模型对否定句解析很差。不写 NOT / 不要 / 没有 / 绝不 / 不是…而是… 等排除句。要强调某状态，用正向高密度堆叠（"完整平实的地面""一动不动""纹丝不动"是正向描述符，可用）。
（例外：极少数模型官方文档明确支持反向锚定段落的，按官方来；不确定就一律正向。）

### 原理 6 ·【音画同步，不写独立音效段】
每个音效内联在它对应的视觉动作紧后面（"抬手砸下，伴随金属撞击的咚一声"），不单开一段集中罗列音效。不要背景音乐就在 prompt 里明写"仅环境音与动作音，不配 BGM"。

### 原理 7 ·【外观靠图锚定，prompt 不复述】
图生视频时，角色外观特征（独眼/疤痕/服装/义肢/发色等）依赖参考图本身。prompt 里**只在需要建立物理逻辑或动作铺垫时**才提外观（例："他抬起机械义肢的左手"——因为左右手材质不同、决定动作），否则只写"这一刻发生什么"。
- 每次出现都重写完整外观 = 错误做法（浪费、且易和图冲突）。
- 注意跨角色污染：别把 A 角色的特征措辞（如"独眼般的"）误用到 B 角色身上，模型会照渲。

### 原理 8 ·【关键表演刻画复用姜导视频表演转译模块】
当镜头承担关键情绪、台词转折、反应镜头、关系变化、痛感/恐惧/羞耻/压抑/撒娇/崩溃/嘴硬/震惊等人物表演任务时，必须先用"姜导表演指导"做一次内部拆解，再把结果翻译进 prompt 正文。普通过场镜头、纯环境镜头、纯动作交代镜头不强行堆微表情。

姜导模块的核心任务：把抽象情绪拆成可见、可演、可拍、可生成的表演结构。它只解决"人物怎么演"，不替代构图、运镜、光线、音效规则。

内部拆解按五步走，分析过程不直接输出，最终只写可见事实：

1. **情绪拆解**：主情绪 + 副情绪 + 隐藏情绪 + 情绪冲突点 + 表演强度。  
   例：小生气 = 表层不满 + 副情绪委屈 + 隐藏动机想被重视 + 冲突点是不想先服软 + 中低强度克制表达。
2. **眼神优先**：先确定看哪里、看多久、是否直视/回避/偷看/下压/放空/上看/侧看，再确定眼神强度是松、稳、紧、锁定还是闪动。人物情绪首先从眼神进入，眼神缺失等于表演缺失。
3. **微表情与面部肌肉**：写眼皮张力、眉部状态、嘴角/嘴唇状态、下颌状态、面部整体张力和呼吸。除非用户明确要求强烈爆发，否则优先使用 30%-60% 面部张力；L4-L5 峰值才进入高张力。
4. **身体语言**：写手指/手掌、肩颈、身体轴线、脚尖或重心、互动对象。身体动作必须服务情绪：想靠近还是远离，想表达还是压住，想求助还是防御。
5. **镜头表演转译**：按景别决定表演重点。特写看眼睛、眉毛、嘴角、呼吸和面部张力；近景看脸部、肩颈和手部小动作；半身看面部、手势、身体轴线；全身看站姿、身体方向、脚尖、距离和重心；背影看肩膀、头部角度、停顿、步伐速度和呼吸感。

写入 prompt 时只保留可见事实，不写心理解释。例：
- "她很委屈" → 眼神微湿但不完全直视对方，眉尾轻压，嘴唇抿住，下唇微收，肩膀略缩，呼吸变浅。
- "他强忍怒气" → 直视对方，下眼睑轻收，眉间紧绷，下颌绷紧，右手指节压在桌沿，肩颈僵住不动。
- "她假装镇定" → 嘴角维持很浅的笑，眼神避开对方半秒又拉回，指尖反复摩擦杯壁，身体轴线保持直立，呼吸略浅。
- "嘴硬但在乎" → 眼神故意避开对方，余光短暂扫回，嘴角压住，身体侧开一点，脚尖仍朝向对方。
- "震惊但不是恐惧" → 上眼皮抬起，眼睛睁大但有明确焦点，嘴微张，吸气停住，头部轻微后仰后僵住半拍。

姜导表演质检：
- 是否有主情绪、副情绪、隐藏动机和情绪冲突点？
- 眼神方向和眼神强度是否明确？
- 眉、嘴、下颌、面部张力、呼吸是否具体？
- 手部动作、肩颈状态、身体轴线、脚尖或重心是否服务情绪？
- 景别是否匹配表演强度？特写是否足够微，半身是否补手势，全身是否补空间关系，背影是否补停顿和肩颈？
- 是否避免只写"开心/难过/生气/震惊/复杂/有张力"等抽象情绪词？
- 是否避免所有人物使用同一种微笑、同一种震惊、同一种委屈？
- 是否可演、可拍、可生成、可检查？

禁忌：不写"复杂的眼神""情绪很到位""自然流露""表情丰富"；不把"生气"只写成瞪眼；不把"震惊"默认写成恐怖脸；不把"认真"写成空洞皱眉；不让表演与人物处境、互动对象、镜头景别脱节。

### 原理 9 ·【光影必须落到具体空间与人物脸上】
光影不能只写"电影感光影""高级光影""氛围感强"。GLOBAL STYLE LOCK 负责声明全局质感与光影基调；每个镜头只补本镜头实际可见的光从哪里来、照到哪里、哪里暗下去、阴影如何落在主体和空间里。光影是稳定画面质感、人物状态和空间连续性的关键锚点。

每个镜头至少写清四项：
- **主光方向**：来自画面左侧/右侧/背后/上方/门缝/窗外/火光/屏幕等。
- **主体受光**：脸部、眼睛、肩颈、手、轮廓边缘哪一侧被照亮，哪一侧压暗。
- **阴影落点**：阴影落在墙面、地面、桌面、衣褶、脸颊、眼窝或背景哪个区域。
- **明暗反差**：低反差柔光 / 高反差硬光 / 半张脸入暗 / 轮廓光切出边缘 / 背景压暗突出主体。

光影写法要服务本镜头，不从上一段照搬，也不重复全局质感长句。例：
- "左侧窗外冷光斜切进来，照亮她右半张脸和手背，另一侧脸落入浅灰阴影，墙面留下窗格斜影。"
- "背后火光勾出他的肩线和侧脸轮廓，正面被压暗，只露出眼窝下方一小片暗红反光。"
- "头顶窄束硬光压在桌面中央，手指和茶杯投下短而清晰的阴影，背景人物全部沉在暗部。"

禁忌：不写"光影好看/质感高级/氛围拉满"；不在每镜重复同一句全局光影；全局已经声明的胶片、低饱和、空气感、色彩基调不在每镜机械复述；不写会强化塑料感的汗水、油光、反光等材质词。若项目明确需要雨夜、潮气、水迹等环境状态，可以写可见的湿痕、水迹、雾气、雨线。

### 原理 10 ·【风格锁只设一个主参考，其他都翻译成画面指令】
GLOBAL STYLE LOCK 只能保留一个主参考作品，用来锚定整体色调、饱和度、光影质感三个维度。多个参考作品同时写进全局风格锁，模型往往做加权平均，不会稳定融合，容易生成"风格平均脸"，也会导致不同镜头随机偏向不同参考。

其他参考作品的有效信息必须转成具体画面指令：
- "暗部保留细节，不死黑压死"替代第二部作品名。
- "前中后景有明确纵深层次"替代第三部作品名。
- "画面克制不堆砌，留有呼吸空间"替代第四部作品名。

参考作品名只在镜头层级按需使用，且只锚定某一个具体维度（如"参考库布里克式正面居中构图"），不能在 GLOBAL STYLE LOCK 中堆作品名。判断原则：具体画面指令比参考作品名更稳定；作品名只在某个视觉特征很难用文字精确描述时使用。

### 原理 11 ·【每个镜头必须有自然语言节奏锚】
分镜分析里的情绪强度和速度信息，必须渗透进每个镜头的 `[镜头风格与节奏]` 行。可以在内部用 L1-L5 判断强度，但最终交付给视频模型时，必须翻译成自然语言，不能把 L1/L2/L3/L4/L5 这类内部码写进 prompt。

自然语言节奏锚要告诉模型四件事：镜头运动快慢、人物动作幅度、声音密度、情绪在本组里的位置。它会影响运镜速度、动作幅度、音效密度和画面内部运动频率。

正确写法：
- `本镜是环境建立，节奏缓慢、压低、安静，画面主要依靠雨点和烛火内部运动。`
- `前半段极慢，台词断住后，手指、运镜、呼吸同时停止，画面进入冻结感。`
- `全片第一个情绪峰值；固定镜头不动，让孩子的崩溃在画面里突然爆开。表演使用短句，动作密度高，声音从屏息直接破裂。`

建立镜头要写清"慢、低密度、动作小、用于建立"；峰值镜头要写清"骤停/爆裂/动作密度高/声音突然冲出"；余韵镜头要写清"运动减少、声音回落、画面留白"。如果建立镜头和峰值镜头读起来语感差不多，说明节奏没有写进去。

### 原理 12 ·【运镜必须有动机，固定镜头必须主动等待】
运镜不只写"怎么动"，还要写"为什么动"和"跟什么同步"。每个运镜段至少包含三要素：
- **运动类型**：dolly in / dolly out / lateral truck / handheld / Micro Move / pan / tilt / crane / tracking 等。
- **同步关系**：跟随谁、等待什么、在哪个动作或表情点停止。
- **速度与加减速**：匀速、渐快、骤停、约几秒推进多少画面距离。

错误：`极缓 dolly in。`

正确：`极缓 dolly in，速度与她的步伐严格同步，约3秒内推进画面景深约三分之一，匀速无加减速。`

固定镜头不能只写"固定"或"绝对静止"，必须写成主动等待：
- `固定镜头——镜头不动，等待雨点与烛火在画面里完成动作。`
- `固定镜头——镜头不动，等她开口。`
- `固定镜头——镜头不动，让崩溃发生在画面里。`

固定镜头的运动责任转移给画面内部，必须单独写"画面内部运动"：烛火摆动频率、雨点节奏、人物呼吸带动的胸腔起伏、泪水滑落速度、衣料和发丝的被动运动。

### 原理 13 ·【峰值、轴心动作和转场必须有结构权重】
情绪峰值镜头（L4-L5）不能沿用建立镜头的长句语感。峰值镜头动作段和表演段必须短句化，动词密度提高，每句只写一个动作机制和一个结果。可以使用"肌肉群/身体部位——可见结果"结构：
- `嘴角向下坠落。`
- `下巴皮肤剧烈起皱。`
- `上眼睑内侧抬升，泪水溢出。`
- `眼裂开到最大，瞳孔放大，视线茫然。`
- `嘴巴张开。`

情绪轴心动作是整段情绪的引爆点，不能混在普通动作序列里，必须独立成段，用短句和重复强调钉死状态：

```
手指缓缓从左向右移动，指节弯曲，触到发丝边缘——

然后，完全停止。

手指悬在发丝上方约一厘米处。不松弛，不颤抖，只是停止运动。
运镜在手停的同一帧同步静止。这个悬空静止状态维持到切镜。
```

转场必须按镜头关系判断。HARD CUT 是默认项，但不是总是正确：
- 同一情绪内部流动 → `DISSOLVE——{叠化对位元素}在叠化中衔接，{持续的声音}不断。`
- 视角或空间信息切换 → `HARD CUT`
- 形状或动作匹配 → `MATCH CUT——以{匹配元素}为对位基准，切入{下一镜描述}。`
- 情绪爆破点 → `HARD CUT——{触发动作}的同一帧切入，不等待，情绪爆破切。`
- 神性/留白段 → 极缓 `DISSOLVE`，或用长镜头内部完成，不切。

## 第二部分：严格输出模板（每条 prompt 必须遵守）

所有视频 prompt 的交付格式必须使用下列骨架。规则层可以内部分析，但最终输出必须按此模板落地；不得改标题层级、不得省略 `[GLOBAL STYLE LOCK]`、不得把镜头内容改成字段表格。

### 默认交付方式：大分镜组独立可粘贴单元

### 项目级风格锁示例：东方医毒权谋 / 雨夜内宅 / 低魔克制

当项目接近《凤隐药尘》这类"东方王朝 + 内宅命案 + 医毒权谋 + 雨夜烛火 + 女性复仇 + 低魔克制"题材时，GLOBAL STYLE LOCK 只保留一个主参考作品，不再只写泛泛的"低饱和古风"，也不堆多个参考作品名。

推荐主参考：
- **《影》**：只用于锁定雨夜水墨冷灰、物理降饱和、压抑权谋、东方水墨质感。

其他参考转译为具体画面指令，不再写作品名：
- 暗部保留细节，不死黑压死；宫廷/内宅暗部仍能看清人物轮廓、衣褶和空间层次。
- 前景、中景、后景有明确纵深层次；人物、门框、灯盏、廊柱、雨幕之间有清晰距离关系。
- 画面克制不堆砌，留有呼吸空间；低对白、低动作幅度，非奇观化动作，低魔段只出现小范围微弱显现。

不推荐作为主风格参考：
- **《延禧攻略》式高明度宫斗配色**：容易把画面带向明亮、装饰性、爽剧化，不适合雨夜毒局。
- **仙侠奇观视觉**：会污染低魔医毒逻辑，导致药灵护心变成法阵、能量爆发或大面积发光。

可直接复用的 GLOBAL STYLE LOCK：

```
[STYLE LOCK]

画幅16:9横版，35mm胶片颗粒质感，柔和胶片高光，低对比冷灰蓝主调；室内以烛火、灯盏、炉火作为小范围暖色动机光，室外以雨夜冷光、泥水反光、雾气和暗山轮廓建立压迫感。低魔药灵只呈现极淡青灰生机，贴近药草、血滴、心口边缘，小范围、低亮度、无爆发、无法阵。

暗部保留细节，不死黑压死；前景、中景、后景有明确纵深层次；画面克制不堆砌，留有呼吸空间；低对白、低动作幅度，非奇观化动作。

仅环境音与动作音，不配BGM；声音以雨声、烛芯、衣料、瓷器、脚步、火声、呼吸和低声对白推动节奏。
```

一句话风格锚：

```
主参考《影》的雨夜水墨冷灰；暗部保留细节，前中后景纵深明确，画面克制留白，低魔只做小范围微弱显现。
```

### 每个镜头段落必须包含
1. **构图**（精确）：主体 @ID、景别、画面位置、占画面比例、前中后景各是什么、当前视角能看到的细节（上半身近景就只写上半身，正面就不写背面）。
2. **镜头风格与节奏**（必须）：每个镜头标题下写 `[镜头风格与节奏]`，用自然语言说明本镜头的运动速度、动作幅度、声音密度、情绪位置和是否骤停/爆发；不得写 L1/L2/L3/L4/L5 这类内部分析码。
3. **镜头状态**（精确）：是否固定；如固定，必须写"主动等待"意图；如运动，写明运动类型（dolly in/back、lateral truck、pan、tilt、crane、tracking、whip pan、handheld、Micro Move 等）+ 同步关系 + 速度 + 方向 + 起点终点 + 是否加减速。
4. **画面内部运动**（固定镜头必须）：烛火、雨点、呼吸、泪水、衣料、发丝、烟尘等运动的频率、幅度、方向和速度。
5. **光影效果**（具体）：主光方向、主体受光部位、阴影落点、明暗反差；每镜按实际画面重写，不照搬全局光影。
6. **人物动作**：动词 + 目标，写"做了什么"不写"心情/动机"；心情翻译成可见行为。
7. **关键表演刻画**（按需）：关键情绪、台词转折、反应镜头必须复用姜导视频表演转译模块，先拆主情绪、副情绪、隐藏动机、情绪冲突点和表演强度，再写眼神方向/强度、眉、嘴、下颌、面部张力、呼吸、手势、肩颈、身体轴线、脚尖或重心；非关键镜头不堆。
8. **峰值镜头短句化**（L4-L5 必须）：动作段和表演段提高动词密度，每句只写一个动作机制和一个可见结果。
9. **情绪轴心动作**（按需）：分镜中标注为核心引爆点、骤停、凝固、崩溃、倒下、触碰前停住等动作，必须独立成段，不能混在普通动作序列里。
10. **台词**（如有）：引号包裹内容 + 语气描述 + 与动作/表情的同步关系（说到"X"的瞬间眼神移开、嘴角压住、手指收紧等）。
11. **音效**：每个动作紧跟对应音效，中文描述 + 必要时英文术语锚定（如"伴随 heavy ground stomp 沉重的脚步闷响"）。
12. **本镜头边界**（按需）：只固定本镜头实际需要避免跑偏的边界；优先用正向画面描述，确需排除时放在镜头末尾，不能压过主体画面。
13. **切镜/转场方式**（多镜头时）：镜头之间必须写明 HARD CUT / DISSOLVE / MATCH CUT / WHIP PAN / 遮罩转场等。每次使用 HARD CUT 前先判断这里需要切断感还是流动感；非 HARD CUT 必须补一句说明转场如何发生，情绪爆破 HARD CUT 必须写触发帧。

### 多镜头处理
- 一个 clip 内多镜头 → 注明切镜/转场方式（HARD CUT / DISSOLVE / MATCH CUT / WHIP PAN / 遮罩转场等），新镜头**另起段落**重走全流程。
- 转场判断：同一情绪内部流动用 `DISSOLVE`；视角或空间信息切换用 `HARD CUT`；形状或动作存在匹配关系用 `MATCH CUT`；情绪爆破点用带触发帧说明的 `HARD CUT`；神性/留白段优先用极缓 `DISSOLVE` 或长镜头内部完成。
- HARD CUT 可直接写 `HARD CUT`；情绪爆破点写 `HARD CUT——{触发动作}的同一帧切入，不等待，情绪爆破切。`；DISSOLVE 必须说明叠化对位元素和持续声音；MATCH CUT 必须说明相似形状、动作或构图如何衔接；WHIP PAN 必须说明甩动方向和下一镜承接方向；遮罩转场必须说明由哪个前景物、人物身体、门框、袖摆、黑影等遮满画面后切入下一镜。
- 不要用"首先…然后…然后…"的列表式分段，会被模型误读成切镜。
- 大分镜组标题、`[GLOBAL STYLE LOCK]`、`[本组资产图声明]`、`[本组整体节奏说明]`、镜头编号、`[镜头风格与节奏]`、正文段落顺序必须保持一致，便于批量投喂、版本管理和回溯修改；资产绑定区以大分镜组级统一声明为主，单镜只在新增资产或状态变化时补充。

|---|---|---|

## 第四部分：交付前强制三步流程

**阶段1 · 极简骨架**：按元模板写，每个镜头连续顺叙、不分子段。

**阶段2 · 对照查漏**：逐项检查构图四要素是否写满、用户明确要的关键信息有没有漏。漏了补。

**阶段3 · 污染删除 pass**（逐句自问）：
- 输出骨架是否严格符合：大分镜标题 `{编号}｜{标题}｜{开始时间}-{结束时间}｜总时长{秒数}s` → `[GLOBAL STYLE LOCK]` → `[本组资产图声明]` → `[本组整体节奏说明]` → 镜头标题 → `[镜头风格与节奏]` → 画面 → 光影 → 运镜 → 画面内部运动（固定镜头必须）→ 动作/表演/台词 → 音效 → 本镜头边界 → 切镜/转场方式？
- GLOBAL STYLE LOCK 是否只有一个主参考作品？其他参考是否已转化为具体画面指令？
- 每个大分镜组是否自带完整 `[GLOBAL STYLE LOCK]`，不需要用户另找全剧风格锁？

## 第五部分：通用约定
- prompt 正文用目标语言（如中文），专业术语保留英文（dolly in / tracking shot / bullet time / high speed slow motion / lateral truck / 导演名等）。
- 所有 prompt 以纯文本文件（.txt）交付，便于直接投喂模型与版本管理。
- 反"3D/AI 塑料感"靠胶片质感、纪录片质感、自然光、结构性瑕疵实现，**不写汗水/油光/湿润/反光**等材质词（这些反而强化塑料感）。
- 不默认设字数目标；如需控字数，必须用脚本精确计数，不靠估算。
- 资产绑定以场景/段落级统一声明为主，不要求每个镜头都有占位符；只有首次出现、状态资产变化、新增角色/道具/场景、或本镜头需要特殊图锚时才补充。
- 一个角色/场景的尺度若模型容易猜错，在资产绑定区的 @ID 锚定行里直接给绝对尺度参照（"约一头犀牛大小""足球大小"）。
- 关键人物表演刻画复用"姜导视频表演转译模块"：先内部拆主情绪、副情绪、隐藏动机、情绪冲突点和表演强度，再只把可见的眼神方向/强度、眼皮、眉、嘴、下颌、面部张力、呼吸、手势、肩颈、身体轴线、脚尖或重心写入 prompt。
- 全局声明整体质感与光影基调；每镜只写具体光影位置关系：光从哪里来、照亮主体哪里、阴影落在哪里、明暗反差如何服务情绪和空间。

### 一句话总纲
**把每个画面钉死到抽10次不跑偏，把每个"饱满/痛苦/自然"拆成模型能解析的具象层，把 GLOBAL STYLE LOCK 收束到一个主参考，把每个镜头写出节奏锚、主动运镜、具体光影和必要的画面内部运动，把关键人物表情交给姜导视频表演转译模块拆成主副情绪、隐藏动机、眼神、眉嘴下颌、呼吸、手势、肩颈和身体轴线，把峰值与轴心动作单独加权，把转场按情绪关系判断，把每句口语比喻翻译成视觉事实——其余的，正向描述、音画同步、外观靠图、交付前三步过一遍。**

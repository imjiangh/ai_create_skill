# 视频提示词编排

作为 S4 阶段唯一编排器。详细写法和固定输出骨架读取 [`通用视频Prompt规则与元模板.md`](../通用视频Prompt规则与元模板.md)；总控模式下把该文件作为专业规则，不执行其中任何重复开场或独立会话控制。

## 输入契约

正式生产至少需要：

- `P01@locked`：平台、画幅、时长和项目风格。
- `S01@locked`：剧情事实与台词属性。
- `A01@locked` 及本镜所需 A02–A05：资产 ID、文件和图槽。
- `B01@locked`：镜号、时码、构图、调度、摄影机、表演、声音、衔接。
- `B02@locked`：每镜入口/出口连续性状态。

缺少锁定资产时可写 `DRAFT`，但不能交付为最终 Prompt 包。

## 内部流程

### 1. 建立 V01 项目风格锁

从 P01、资产包和 B01 提取唯一主风格、画幅、质感、光色、摄影运动、时间连续性和禁止漂移项。V01 只写项目级稳定规则，不重复每镜内容。

### 2. 建立生成映射

为每个 B01 镜头记录：镜号、所属大分镜组、目标时长、成片时长、资产图槽、首帧/尾帧依赖、模型/模式、前镜出态、后镜入态。

### 3. 判断 Prompt 容器

- 默认按大分镜组输出独立可粘贴单元。
- 15 秒只在平台或交付链明确要求时作为硬容器。
- 动作、空间或状态无法在一个 clip 稳定完成时拆镜；拆分必须回写映射，不擅自改变 B01 的叙事顺序。

### 4. 写逐镜 Prompt

严格使用通用规则的固定骨架。每镜必须具体化：主体位置与占比、前中后景、朝向、节奏、光影落点、运镜动机与终点、画面内部运动、动作相位、表演、台词/声音、正向边界和转场。

外观由资产图锚定，Prompt 引用资产 ID/图槽，不重复创作人物长相。心理词、比喻和抽象形容词必须翻译为可见可听事实。

### 5. 动作密度与模型风险审计

检查每个 clip 的主体数、动作链、镜头运动、口型、VFX 交互和转场负荷。风险过高时优先：减少同段任务、拆动作相位、使用首尾帧或参考图槽、把后期效果分层；不得靠堆负向词解决。

### 6. 连续性核对

逐镜比对 B02：人物位置、朝向、手持物、服装/伤妆、场景状态、光向、VFX 状态、运动方向、焦点和声音桥。发现 B01/B02 冲突时回 S3，不在 Prompt 内偷偷修正。

### 7. 形成 V03 生成清单

每个 Prompt 单元记录文件名、镜号范围、图槽、版本、目标时长、模型参数、依赖、状态和重试说明。重试只修改失败因素，保留成功锁。

## S4 闸门

- V01、V02、V03 的版本和引用一致。
- 每个 B01 镜头都有且只有一个明确的生成去向。
- 总时长、顺序、台词和转场与 B01 一致。
- 所有图槽都能追溯到 A01，所有状态都能追溯到 B02。
- Prompt 不发明剧情、人物外观、道具或 VFX 系统。
- 每个单元可独立粘贴，不依赖聊天里的隐含上下文。
- 没有未决 P0/P1 模型执行或连续性问题。

## 返工路由

- Prompt 解析、动作密度或图槽声明问题：S4 内修复。
- 镜头切分、轴线、时长或覆盖问题：回 S3。
- 外观、空间、道具或 VFX 定义问题：回 S2。
- 剧情事实、台词或人物动机问题：回 S1。

## 交接输出

```text
V01：项目风格锁路径 + 版本 + locked
V02：Prompt 包目录/文件 + 镜号范围 + 版本 + locked
V03：生成清单路径 + 版本 + locked
依赖摘要：资产 ID/图槽 + B01/B02 版本
已接受风险：镜头 + 风险 + 处理方式
执行顺序：唯一推荐的批次顺序
```
